## 为什么需要结构体

假设你要写一个程序，记录三个学生的姓名、年龄、分数。最朴素的写法：

```cpp
std::string name1, name2, name3;
int age1, age2, age3;
double score1, score2, score3;
```

学生数量一多，谁是谁的就乱了——`name1` 和 `age2` 配错了都没人发现。改成数组也好不到哪去：

```cpp
std::string names[3];
int ages[3];
double scores[3];
```

三个数组各管一段，下标对错就出 bug。**真正的问题是**：姓名、年龄、分数明明是"一个学生"的三件事，却被拆成三处保管。

**结构体（struct）** 就是解决这个的——把"属于同一个东西的数据"打包成一个新类型：

```cpp
struct Student {
    std::string name;
    int age;
    double score;
};

Student a;
a.name = "亚柯洲";
a.age = 13;
a.score = 95.5;
```

一行声明 `Student a;`，三个字段就一起在了——再也不会配错。

## 基本语法

### 声明结构体

```cpp
struct 结构体名 {
    类型 字段1;
    类型 字段2;
    // ...
};                      // 注意分号！
```

例子：

```cpp
struct Point {
    double x;
    double y;
};

struct Book {
    std::string title;
    std::string author;
    int pages;
    double price;
};
```

::: callout 别忘了最后那个分号
`};` 不是 `}`——结构体定义末尾**必须**有分号。漏了它编译器会给一堆莫名其妙的报错，是新手 top 5 错误之一。
:::

### 创建变量

```cpp
struct Point {
    double x;
    double y;
};

int main() {
    Point p1;                  // 字段没初始化，里面是垃圾值
    Point p2{};                // 字段全部清零（推荐）
    Point p3{3.0, 4.0};        // 按字段顺序初始化
    Point p4 = {3.0, 4.0};     // 等价写法

    p1.x = 1.0;                // 用 . 访问字段
    p1.y = 2.0;
    std::cout << p1.x << ", " << p1.y << "\n";
}
```

最常见的初始化方式是 `Point p{3.0, 4.0}`——**大括号里按字段定义的顺序**填值。

### 字段默认值（C++11 起）

```cpp
struct Point {
    double x = 0.0;
    double y = 0.0;
};

Point p;            // x=0, y=0，不是垃圾
Point q{3.0};       // x=3, y=0
Point r{3.0, 4.0};  // x=3, y=4
```

写默认值之后，**没指定的字段会用默认值**——再也不用担心忘了初始化。**养成给字段写默认值的习惯**。

### 指定初始化（C++20）

C++20 加了"按字段名指定"的写法，类似 Python 的关键字参数：

```cpp
Point p{.x = 3.0, .y = 4.0};
Book b{
    .title = "C++ Primer",
    .author = "Lippman",
    .pages = 1000,
    .price = 99.0
};
```

字段一多时**比按顺序写清楚得多**——一眼看出哪个值给了哪个字段。要 C++20 编译器：`g++ -std=c++20`。

## 访问字段：`.` 还是 `->`

- 直接拿对象 → 用 `.`：`p.x`
- 拿对象的指针 → 用 `->`：`pp->x`（等价于 `(*pp).x`）

```cpp
Point p{3.0, 4.0};
Point* pp = &p;

std::cout << p.x << "\n";        // 用 .
std::cout << pp->x << "\n";      // 用 ->
std::cout << (*pp).x << "\n";    // 等价但难看，几乎没人这么写
```

这个区别上一篇（OOP）讲过，结构体里完全一样。

## 嵌套结构体

结构体的字段也可以是另一个结构体：

```cpp
struct Point {
    double x;
    double y;
};

struct Rectangle {
    Point top_left;
    Point bottom_right;
};

int main() {
    Rectangle r{{0, 10}, {20, 0}};      // 嵌套大括号

    std::cout << r.top_left.x << "\n";   // 0
    std::cout << r.bottom_right.x << "\n"; // 20

    // 也能一步步赋值
    Rectangle r2;
    r2.top_left.x = 0;
    r2.top_left.y = 10;
    r2.bottom_right.x = 20;
    r2.bottom_right.y = 0;
}
```

嵌套**没有层数限制**——但实际写代码超过两层就不容易读了。**别为了嵌套而嵌套**。

## 结构体做函数参数

### 传值：函数拿到的是拷贝

```cpp
void print_point(Point p) {        // 传值
    std::cout << p.x << ", " << p.y << "\n";
    p.x = 999;                     // 改的是拷贝，不影响外面
}

Point p{3.0, 4.0};
print_point(p);
std::cout << p.x << "\n";          // 还是 3.0
```

**问题**：结构体大的时候（比如有几个 `std::string` 字段），拷贝有成本。

### 传 const 引用：又快又安全

```cpp
void print_point(const Point& p) {     // const 引用
    std::cout << p.x << ", " << p.y << "\n";
    // p.x = 999;                      // 编译错：不能改 const
}
```

**只读不改、避免拷贝**——这是绝大多数情况下的最佳写法。p005（函数进阶）那篇讲过，结构体一样适用。

### 传引用：函数能改外面的

```cpp
void scale(Point& p, double factor) {   // 普通引用
    p.x *= factor;
    p.y *= factor;
}

Point p{3.0, 4.0};
scale(p, 2.0);
std::cout << p.x << ", " << p.y << "\n";   // 6, 8
```

**只在确实要改外面的对象时才用普通引用**。

::: callout 一条黄金规则
- 字段都是基础类型（`int`、`double`）的小结构体（不超过 16 字节）→ **传值**也行。
- 大结构体或含有 `std::string`、`std::vector` 字段 → **`const T&` 只读**或 **`T&` 要改**。

入门阶段记不清就一律 `const T&`，吃不了亏。
:::

## 结构体做返回值

返回结构体很常见——比如一个函数想"一次返回多个值"：

```cpp
struct DivResult {
    int quotient;       // 商
    int remainder;      // 余数
};

DivResult divide(int a, int b) {
    return {a / b, a % b};      // 直接用大括号构造
}

int main() {
    DivResult r = divide(17, 5);
    std::cout << "商 " << r.quotient << "，余 " << r.remainder << "\n";
    // 商 3，余 2
}
```

或者配合**结构化绑定**（p008 STL 那篇讲过）一行拆开：

```cpp
auto [q, r] = divide(17, 5);
std::cout << q << " " << r << "\n";
```

写起来像 Python 的多返回值，**真正的 C++ 多返回值就是这么实现的**。

## 结构体数组 / vector

最经典的用法——一组结构体放进 `std::vector`：

```cpp
#include <iostream>
#include <vector>
#include <string>

struct Student {
    std::string name;
    int age;
    double score;
};

int main() {
    std::vector<Student> students;
    students.push_back({"亚柯洲", 13, 95.5});
    students.push_back({"小明", 12, 88.0});
    students.push_back({"小红", 13, 92.5});

    // 遍历打印
    for (const auto& s : students) {
        std::cout << s.name << " " << s.age << " " << s.score << "\n";
    }
}
```

**这是入门阶段最值得掌握的写法**——一行 `std::vector<Student>` 就能管理任意多个学生。

### 排序结构体

`std::sort` 默认按 `<` 排，结构体没有 `<` 怎么办？传一个 lambda 告诉它怎么比：

```cpp
#include <algorithm>

// 按分数从高到低排
std::sort(students.begin(), students.end(),
    [](const Student& a, const Student& b) {
        return a.score > b.score;
    });

for (const auto& s : students) {
    std::cout << s.name << " " << s.score << "\n";
}
// 亚柯洲 95.5
// 小红 92.5
// 小明 88
```

想按其他字段排，改 lambda 里的 `>` / `<` 就行。**lambda 那段就是"两个学生谁排前面"的判断**。

### 查找结构体

```cpp
auto it = std::find_if(students.begin(), students.end(),
    [](const Student& s) { return s.name == "小明"; });

if (it != students.end()) {
    std::cout << "找到了：" << it->name << " " << it->score << "\n";
} else {
    std::cout << "没找到\n";
}
```

注意：`it` 是迭代器（类似指针），用 `->` 访问字段。

## 结构体也能有函数

C++ 里 `struct` 不只是"一堆字段"——它**和 `class` 几乎一样**，可以有函数（成员函数）：

```cpp
struct Point {
    double x;
    double y;

    double distance_from_origin() const {
        return std::sqrt(x * x + y * y);
    }

    void move(double dx, double dy) {
        x += dx;
        y += dy;
    }
};

Point p{3.0, 4.0};
std::cout << p.distance_from_origin() << "\n";   // 5
p.move(1.0, 1.0);
std::cout << p.x << ", " << p.y << "\n";          // 4, 5
```

::: callout struct 和 class 的唯一区别
**默认访问权限不一样**：
- `struct` 默认 `public`（外面能访问字段）。
- `class` 默认 `private`（外面看不见，要写 `public:` 才行）。

约定俗成：
- **纯数据打包** → 用 `struct`（不写 public/private 也方便）。
- **有封装、有规则的复杂类型** → 用 `class`。

这篇全用 `struct`，不写 `public:` 就能直接用——更适合入门阶段。
:::

## 一个综合例子：成绩管理

把上面学的串起来——一个能添加、查找、按分数排序、求平均的小程序：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <string>

struct Student {
    std::string name;
    int age = 0;
    double score = 0.0;

    void print() const {
        std::cout << name << "（" << age << " 岁）：" << score << " 分\n";
    }
};

double average_score(const std::vector<Student>& students) {
    if (students.empty()) return 0.0;
    double sum = 0;
    for (const auto& s : students) sum += s.score;
    return sum / students.size();
}

void sort_by_score_desc(std::vector<Student>& students) {
    std::sort(students.begin(), students.end(),
        [](const Student& a, const Student& b) {
            return a.score > b.score;
        });
}

const Student* find_by_name(const std::vector<Student>& students,
                             const std::string& name) {
    for (const auto& s : students) {
        if (s.name == name) return &s;
    }
    return nullptr;
}

int main() {
    std::vector<Student> students;
    students.push_back({"亚柯洲", 13, 95.5});
    students.push_back({"小明",   12, 88.0});
    students.push_back({"小红",   13, 92.5});
    students.push_back({"小李",   14, 78.0});

    std::cout << "—— 全班 ——\n";
    for (const auto& s : students) s.print();

    std::cout << "\n—— 按分数排序 ——\n";
    sort_by_score_desc(students);
    for (const auto& s : students) s.print();

    std::cout << "\n平均分：" << average_score(students) << "\n";

    std::cout << "\n查找小明：";
    if (auto p = find_by_name(students, "小明")) {
        p->print();
    } else {
        std::cout << "没找到\n";
    }
}
```

短短五十行，覆盖了：结构体定义 + 默认值 + 成员函数 + vector 装结构体 + lambda 排序 + 引用传参 + 指针返回值。**这是写实战 C++ 的"标准重量"**。

## 常见坑

::: callout 1. 结构体定义后没加分号
```cpp
struct Point {
    double x;
    double y;
}                    // ← 这里漏分号

int main() { ... }   // 编译器在这里开始报奇怪的错
```
**`};` 不是 `}`**——记住这一条少踩 80% 的坑。
:::

::: callout 2. 字段没初始化就用
```cpp
struct Point { double x; double y; };
Point p;
std::cout << p.x;     // 垃圾值，不一定是 0
```
**给字段写默认值**：`double x = 0;` 或者用 `Point p{};`（带括号会全部清零）。
:::

::: callout 3. 大括号初始化字段顺序错了
```cpp
struct Student { std::string name; int age; double score; };
Student s{13, "小明", 95.5};      // 编译错：第一项是 int 不能给 string
```
大括号里的值**严格按字段定义顺序**。乱写编译器会报错；如果**类型恰好都对**就更糟——它编得过但语义全错。**字段多时用 C++20 的指定初始化** `{.name="...", .age=13}` 更安全。
:::

::: callout 4. 传大结构体没加引用
```cpp
void print(Student s) { ... }     // 每次调用都拷贝整个 Student
```
有 `std::string` 字段时拷贝挺贵的。**`const Student&` 默认更好**。
:::

::: callout 5. 想改外面的对象忘了加 `&`
```cpp
void grow(Student s) { s.age++; }   // 改的是拷贝
```
要改原对象，参数必须是 `Student&`（引用）或 `Student*`（指针）。
:::
