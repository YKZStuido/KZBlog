## 从源码到可执行文件

写一个 `.cpp` 文件到能在终端里跑起来，中间发生了四件事：

1. **预处理（Preprocess）**：处理 `#include`、`#define` 等以 `#` 开头的指令。`#include <iostream>` 会被替换成 `iostream` 这个头文件的全部内容。
2. **编译（Compile）**：把预处理后的 C++ 代码翻译成汇编语言。
3. **汇编（Assemble）**：把汇编翻译成机器指令，生成 `.o`（Linux/macOS）或 `.obj`（Windows）目标文件。
4. **链接（Link）**：把你的目标文件和标准库、其他依赖拼成最终的可执行文件。

```
hello.cpp ─► [预处理] ─► [编译] ─► [汇编] ─► hello.o ─► [链接] ─► hello
```

平时用 `g++ hello.cpp -o hello` 一条命令把四步都做了。知道这个分阶段的模型，排错时才明白「为什么报错的不是语法而是链接器」。

::: callout 编译错 vs. 链接错
`undefined reference to 'xxx'` 这类错误不是语法错，是**链接错**——函数声明了但没找到定义。新手很容易被它的位置误导，记住：报错里出现 `ld`、`linker`、`undefined reference`，就往「链接阶段」想。
:::

## 拆解 Hello, World

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, C++!" << std::endl;
    return 0;
}
```

一行一行看：

- `#include <iostream>`：预处理指令，引入标准输入输出库。`iostream` 里有 `cin`、`cout` 这些对象。
- `int main() { ... }`：**主函数**，程序从这里开始执行。`int` 表示它返回一个整数。
- `std::cout << "..." << std::endl;`：向标准输出打印一个字符串，然后换行。
- `return 0;`：返回 0 给操作系统，约定俗成表示「程序正常结束」。

### 关于 `std::`

`std` 是 C++ 标准库的**命名空间**（namespace），用来避免同名冲突。`std::cout` 读作「std 下面的 cout」。

有人写教程会加一句 `using namespace std;` 省掉前缀：

```cpp
using namespace std;
cout << "Hello" << endl;
```

**不推荐**。在任何稍微大一点的项目里，这会把标准库几千个名字全倒进当前作用域，极易撞名。**老老实实写 `std::`**，多敲几个字母换一辈子安心。

## 变量与类型

C++ 是**静态类型**语言——每个变量都有固定类型，声明时必须说清楚。

```cpp
#include <iostream>
#include <string>

int main() {
    int age = 13;                   // 整数
    double height = 1.68;           // 浮点数（双精度）
    char grade = 'A';               // 单个字符，注意单引号
    bool is_student = true;         // 布尔值
    std::string name = "Ya Kezhou";    // 字符串

    std::cout << name << " is " << age << " years old." << std::endl;
    std::cout << name << " is " << height << "m and " << grade << "." << std::endl;
    return 0;
}
```

常用基础类型速览：

- `int`：整数，通常 32 位，范围大约 ±21 亿。
- `long long`：更大的整数，至少 64 位。大数运算用它。
- `double`：双精度浮点数，**做小数运算默认用它**，别用 `float`。
- `char`：单个字符，单引号。`'A'` 是字符，`"A"` 是字符串。
- `bool`：`true` 或 `false`。
- `std::string`：字符串，用前要 `#include <string>`。

### `auto`：让编译器帮你推导

写 `std::vector<std::pair<int, int>>::iterator it = v.begin();` 谁都要崩溃。C++11 开始可以写：

```cpp
auto it = v.begin();
```

编译器根据右边的表达式自动推导左边的类型。**初始化必须同时写**，否则编译器没法推。

### 常量：`const` 和 `constexpr`

```cpp
const int MAX_USERS = 100;       // 运行时常量
constexpr int BUFFER = 1024;     // 编译期常量
```

一般写 `const` 就够用；`constexpr` 用于要在编译期就确定的值（比如数组大小）。**能加 const 就加 const**，能让编译器帮你拦住一半的低级错误。

## 输入输出

```cpp
#include <iostream>
#include <string>

int main() {
    std::string name;
    int age;

    std::cout << "名字？";
    std::cin >> name;           // 读取一个不含空格的词

    std::cout << "年龄？";
    std::cin >> age;            // 读取整数

    std::cout << "你好 " << name << "，" << age << " 岁。\n";
    return 0;
}
```

几个要点：

- `>>` 读输入，`<<` 写输出。方向指向**数据流动方向**，记起来不难。
- `"\n"` 是换行符；`std::endl` 也是换行，但会**强制刷新缓冲区**，比 `"\n"` 慢一点。平时用 `"\n"` 就行。
- `std::cin >> name` 遇到空白（空格、Tab、换行）就停。想读一整行要用 `std::getline(std::cin, name)`。

### 混用 `cin >>` 和 `getline` 的坑

```cpp
int n;
std::string line;
std::cin >> n;
std::getline(std::cin, line);   // 直接读到空行！
```

因为 `cin >> n` 读完数字后，**回车还留在缓冲区**，`getline` 直接吃到了它。解决办法：

```cpp
std::cin >> n;
std::cin.ignore();              // 忽略那个换行
std::getline(std::cin, line);
```

## 运算符

### 算术

```cpp
int a = 7, b = 3;
std::cout << a + b << "\n";     // 10
std::cout << a - b << "\n";     // 4
std::cout << a * b << "\n";     // 21
std::cout << a / b << "\n";     // 2   ← 整数除法！
std::cout << a % b << "\n";     // 1   ← 取余
```

**整数除法会直接扔掉小数部分**，`7 / 3` 是 `2` 不是 `2.33`。想要小数结果，至少有一个操作数得是浮点：

```cpp
std::cout << 7 / 3.0 << "\n";   // 2.33333
std::cout << (double)a / b << "\n";  // 2.33333
```

练习题目：[P1001 A+B Problem](https://www.luogu.com.cn/problem/P1001)

### 比较与逻辑

```cpp
a == b    // 相等（注意是两个等号）
a != b    // 不等
a <  b    // 小于
a >= b    // 大于等于
a && b    // 与
a || b    // 或
!a        // 非
```

::: callout 最常见的新手 bug
把 `==` 写成 `=`。
`if (x = 5)` 是**赋值**，不是比较，结果永远为真。打开 `-Wall` 警告就会提醒你这事。
:::

### 自增自减

```cpp
int i = 5;
++i;        // i 变成 6
i++;        // i 变成 7
```

单独使用时 `++i` 和 `i++` 一样；在表达式里前者返回「加完之后的值」，后者返回「加之前的值」。**看得懂就好，别写那种依赖它差别的代码**——难懂，而且容易出 bug。

## 分支

```cpp
int score = 82;

if (score >= 90) {
    std::cout << "A\n";
} else if (score >= 80) {
    std::cout << "B\n";
} else if (score >= 60) {
    std::cout << "C\n";
} else {
    std::cout << "不及格\n";
}
```

多分支又互斥时可以用 `switch`：

```cpp
int day = 3;
switch (day) {
    case 1: std::cout << "周一\n"; break;
    case 2: std::cout << "周二\n"; break;
    case 3: std::cout << "周三\n"; break;
    default: std::cout << "周末吧\n";
}
```

> 每个 `case` 后面必须加 `break`，否则会「掉穿」到下一个 case 继续执行。这是 C++ 沿袭自 C 的历史包袱，记住就好。

## 循环

### for 循环：次数确定

```cpp
for (int i = 0; i < 5; ++i) {
    std::cout << i << " ";     // 0 1 2 3 4
}
```

三段分别是：**初始化** ; **继续循环的条件** ; **每轮结束后执行**。

### while 循环：条件驱动

```cpp
int n = 10;
while (n > 0) {
    std::cout << n << " ";
    n /= 2;                    // 10 5 2 1
}
```

### 范围 for：遍历容器

```cpp
#include <vector>

std::vector<int> nums = {3, 1, 4, 1, 5, 9};
int sum = 0;
for (int x : nums) {
    sum += x;
}
std::cout << "和 = " << sum << "\n";
```

读作「对于 nums 里的每个 x」。现代 C++ 里能用范围 for 就尽量用，比传统下标循环更安全、更短。

### `break` 和 `continue`

- `break`：立刻跳出**当前循环**。
- `continue`：跳过本轮剩下的代码，直接进入下一轮。

```cpp
for (int i = 1; i <= 10; ++i) {
    if (i % 2 == 0) continue;   // 跳过偶数
    if (i > 7) break;           // 到 7 就停
    std::cout << i << " ";      // 1 3 5 7
}
```

## 函数

函数是**把一段可复用的代码打包起来**的基本单位。

```cpp
#include <iostream>

int add(int a, int b) {
    return a + b;
}

bool is_even(int n) {
    return n % 2 == 0;
}

int main() {
    std::cout << add(3, 5) << "\n";         // 8
    std::cout << is_even(10) << "\n";       // 1（true）
    return 0;
}
```

函数定义的三要素：

- **返回类型**（`int`、`bool`、`void` 等，`void` 表示不返回）
- **函数名**（约定小写字母加下划线，或者驼峰，风格统一即可）
- **参数列表**

### 传值 vs. 传引用

```cpp
void grow(int x) { x += 1; }         // 改不了外面
void grow_ref(int& x) { x += 1; }    // 直接改外面的变量

int a = 5;
grow(a);        std::cout << a << "\n";  // 5
grow_ref(a);    std::cout << a << "\n";  // 6
```

- 不加 `&`：**传值**——函数拿到的是一份拷贝，改它不影响外面。
- 加 `&`：**传引用**——等于给外面的变量起了个别名，改它就是改原变量。

传大对象（比如长字符串、大 vector）时，**传引用能避免拷贝**，效率高很多：

```cpp
void print(const std::string& s) {   // const + 引用：只读但不拷贝
    std::cout << s << "\n";
}
```

### 函数声明和定义

大工程里函数往往先在头文件里**声明**，在 `.cpp` 里**定义**：

```cpp
// math.h
int add(int a, int b);     // 声明：只有签名

// math.cpp
int add(int a, int b) {    // 定义：有函数体
    return a + b;
}
```

入门阶段写单文件程序用不上这个区分，了解即可。

## 数组与 vector

### 原始数组

```cpp
int arr[5] = {3, 1, 4, 1, 5};
std::cout << arr[0] << "\n";     // 3
std::cout << arr[4] << "\n";     // 5
```

- 下标**从 0 开始**——长度 5 的数组合法下标是 `0` 到 `4`。
- **越界访问是未定义行为**，不会报错，但可能读到垃圾值或直接崩溃。

### std::vector：会自己长大的数组

原始数组长度固定，实用场景下更常用 `std::vector`：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;
    v.push_back(10);
    v.push_back(20);
    v.push_back(30);

    std::cout << "长度 = " << v.size() << "\n";
    for (int x : v) std::cout << x << " ";
    std::cout << "\n";
    return 0;
}
```

`vector` 的常用操作：

- `v.push_back(x)`：末尾追加。
- `v.size()`：当前元素个数。
- `v[i]`：下标访问，**不做边界检查**。
- `v.at(i)`：带边界检查的下标访问，越界会抛异常。
- `v.empty()`：是否为空。

::: callout vector 是你最好的朋友
入门阶段「需要一个数组」的 99% 场合，都应该写 `std::vector`。它自动管理内存，能动态扩容，API 清爽——没有理由去跟 C 风格数组较劲。
:::

## 一个综合例子

把上面学的串起来——输入 n 个数，输出它们的平均值和最大值：

```cpp
#include <iostream>
#include <vector>
#include <limits>

int main() {
    int n;
    std::cout << "要输入几个数？";
    std::cin >> n;

    std::vector<double> nums;
    nums.reserve(n);

    for (int i = 0; i < n; ++i) {
        double x;
        std::cin >> x;
        nums.push_back(x);
    }

    double sum = 0;
    double max_val = std::numeric_limits<double>::lowest();
    for (double x : nums) {
        sum += x;
        if (x > max_val) max_val = x;
    }

    std::cout << "平均 = " << sum / n << "\n";
    std::cout << "最大 = " << max_val << "\n";
    return 0;
}
```

试着跑一下，再改改——比如也输出最小值，或者算一下每个数减去平均值的偏差。**动手改永远比读十遍印象深**。

## 入门阶段常见陷阱

::: callout 1. 忘记 `#include`
用 `std::string` 没 include `<string>`，用 `std::vector` 没 include `<vector>`。报错通常是「没有这个名字」或「不完整类型」。
:::

::: callout 2. 把 `=` 和 `==` 搞混
`=` 是赋值，`==` 是比较。`if (x == 10)` 对；`if (x = 10)` 恒真且顺手改了 x。
:::

::: callout 3. 数组下标越界
`arr[n]` 在长度为 `n` 的数组里是越界的。合法范围是 `0` 到 `n-1`。
:::

::: callout 4. 忘记变量要初始化
`int x; std::cout << x;` 打印的是**随机值**，不是 0。局部变量**不会自动清零**。
:::

::: callout 5. 整数除法出小数 0
`1 / 2` 在 C++ 里是 `0` 不是 `0.5`。至少让一个操作数变成浮点：`1.0 / 2`。
:::
