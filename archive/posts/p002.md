## 前置说明

本教程适用于**一加**手机，其中大部分内容也适用于大部分 Android 设备，但以下两类机型需特别注意：

- **运行 EMUI 8+ 的华为手机**：官方解锁码申请通道已于 2018 年关闭，Bootloader 无法通过官方途径解锁；且其分区方案不遵循标准 Android 分区，无法使用标准方法获取 Root。
- **三星部分机型**：国行 / 港行 / 欧版等部分型号解锁 Bootloader 时会熔断 KNOX，导致三星支付等安全功能失效；首次安装 Magisk 需完整数据擦除及额外操作，且无法进行 OTA 更新。美版（型号尾字母为 U/U1 等）从未支持解锁 Bootloader。

### 必备工具

- **platform-tools**（包含 `adb` 与 `fastboot`）：Google 官方提供，解压目录记为 `C:\ADB`。
- **USB 数据线**：建议使用原装线，劣质线可能导致 Fastboot 下断连。
- **手机驱动**：高通 / 联发科 / 三星机型在 EDL、Bootrom、Download 模式下均需安装对应驱动。

### 关键术语

- **Bootloader**：引导加载程序，控制设备启动流程并校验分区合法性
- **Fastboot**：Bootloader 暴露出的刷写协议
- **Recovery**：独立于系统的恢复环境，用于刷机、清除数据等
- **EDL / Firehose**：高通紧急下载模式；Firehose 是在 EDL（9008）模式下运行的通信协议
- **A/B 分区**：双槽位机制，支持无缝 OTA
- **Ramdisk**：初始内存盘，存放于 `boot.img` 或 `init_boot.img` 中
- **AVB / VBMeta**：Android Verified Boot 启动校验
- **GKI / KMI**：通用内核镜像 / 内核模块接口，Android 12+ 新机型普遍采用

## 解锁 Bootloader（一加）

> 请备份手机中的所有数据。无论是解锁还是回锁 Bootloader，都会擦除手机中的所有数据。

1. 打开**设置 → 关于本机 → 版本信息**，连续点击「版本号」，直到弹出「已启用开发者选项」。
2. 回到设置首页，打开**系统与更新 → 开发者选项**，启用 **OEM 解锁** 和 **USB 调试**。
3. 使用原装数据线把手机与电脑连接，选择「传输文件」模式。
4. 按 Win + R 打开运行，输入 `cmd` 回车。
5. 切换到 platform-tools 目录：

```bat
cd /d C:\ADB
```

6. 首次运行需要在手机上授权 ADB 调试：

```bat
adb devices
```

再次运行 `adb devices`，直到输出形如 `xxxxxx device`。

7. 重启到 Bootloader：

```bat
adb reboot bootloader
```

8. 执行解锁命令（部分较新的一加 / OPPO / Realme 机型可能需要使用 `fastboot oem unlock` 或官方解锁工具）：

```bat
fastboot flashing unlock
```

9. 在手机上使用音量 +/- 选择 `UNLOCK THE BOOTLOADER`，按电源键确认。
10. 等待开机清理数据。部分机型开机时会短暂显示黄字（Orange State）警告即为成功；新机型可能已不再显示此警告。

## 进入 Fastboot / Recovery 模式

### Fastboot 模式

开机状态下：

```bat
adb reboot bootloader
```

关机状态下：同时按住**电源键**与**音量 −**，看到 Fastboot 图标后松开所有按键。

### Recovery 模式

开机状态下：

```bat
adb reboot recovery
```

从 Fastboot 进入：

```bat
fastboot reboot recovery
```

或在 Fastboot 界面用音量 +/- 选择 `Recovery Mode`，按电源键确认。

## 获取 boot.img / init_boot.img

### 有官方 ROM 包的机型（一加、小米 / 红米、Pixel 等）

前往对应 ROM 下载站，下载与当前机型、系统版本一致的线刷包或卡刷包：

- **线刷包**：解压后进入 `images/` 文件夹，找到 `boot.img` 或 `init_boot.img`。
- **卡刷包**：解压后使用 `payload-dumper-go` 或 MT 管理器解包 `payload.bin`，提取出 `boot.img` 和 `init_boot.img`。

### 无官方 ROM 包的机型（工程机、山寨机等）

**运行 userdebug 系统**（例如 LineageOS GSI）：userdebug 系统命令行自带 Root 权限，可直接提取和刷入分区。

1. 进入 ADB Shell，输入 `su`；提示符从 `$` 变为 `#` 表示已获得 Root。
2. 使用 `dd` 提取分区：

```bash
cd /dev/block/platform
ls
cd by-name
ls -l
dd if=<分区路径> of=/sdcard/<分区名>.img
```

**非 userdebug 系统（已解锁 Bootloader）**：

1. 安装 **Treble Check**，确认手机支持 Treble 标准且已解锁 Bootloader。
2. 安装 **DSU Sideloader**，激活后下载当前 Android 版本的 GSI（不带 GMS 的 userdebug 版本）并侧载。
3. 使用通知栏的「重启」按钮进入 GSI 系统，再按上文 userdebug 的方法提取分区。

**支持深度刷机模式的机型**：

- **一加（高通）**：使用 Oplus EDL Tool 配合深度刷机文件进入 Firehose 模式，读取分区表后提取、刷入。深度刷机免授权整合：`https://pan.yakezhou.asia/s/70Iw`（如果你的手机可以进入深度刷机模式，请尽量不要更新系统，防止厂商熔断！）
- **其他高通机型**：进入 EDL（9008）模式，使用 QPT 工具回读分区，Patch 后再刷写回设备。
- **联发科**：进入 Bootrom 模式，使用 SP Flash Tool 回读分区，Patch 后再刷写回设备。

## 刷入 Magisk / KernelSU / APatch

### 刷入 Magisk

Magisk 通过修补 `boot.img` 或 `init_boot.img` 中的 Ramdisk，由 MagiskInit 加载 SystemLess，实现不修改 `system` 分区即可获取 Root。

> 首发 Android 13（含）及以上的设备，通常采用 GKI 方案且内核版本 ≥ 5.10，Ramdisk 存放于 `init_boot.img`，应修补 `init_boot.img`；其余设备修补 `boot.img`。

**方案 1：修补现有 Ramdisk 后刷入**

1. 获取当前系统对应的 `boot.img` 或 `init_boot.img`。
2. 使用 Magisk 管理端修补镜像。
3. 将修补后的镜像通过 Fastboot 刷回设备。

**方案 2：第三方 Recovery 卡刷**

需要先安装 TWRP、OrangeFox 等第三方 Recovery，并使用较新版本的 Magisk 安装包卡刷，重启后安装管理端。

### 刷入 KernelSU 或衍生版本

KernelSU 通过自带 KernelSU 的 GKI 内核获取 Root，或修补 `boot.img` 使用 LKM 模式。

**方案 1：管理端修补 boot.img**——获取当前系统的 `boot.img`，使用 KernelSU 管理端修补后刷入。

**方案 2：刷入通用内核镜像（GKI）**——刷入 KernelSU 提供的通用内核镜像获取 Root。需满足：KMI 版本与当前设备一致；Android 安全补丁时间大于或等于当前设备。

**方案 3：AnyKernel3 刷机包**——使用内核刷写工具（如 Kernel Flasher）刷入对应内核版本的 AnyKernel3 刷机包。

> 对于所有方案，如果设备支持 `fastboot boot`，可先执行 `fastboot boot boot.img` 试引导。出现异常重启一次即可开机。

### 刷入 APatch

APatch 源自 KernelSU，允许任意代码注入内核，适用于大部分 Android 手机，不限于 GKI 内核设备，支持内核版本约为 **3.18 ~ 6.1**。

判断是否支持修补——在终端以 Root 权限执行：

```bash
zcat /proc/config.gz | grep CONFIG_KALLSYMS
```

完整支持输出：`CONFIG_KALLSYMS=y` 且 `CONFIG_KALLSYMS_ALL=y`；初步支持：`CONFIG_KALLSYMS=y` 且 `CONFIG_KALLSYMS_ALL=n`。

> **超级密钥**必须使用强密码，禁止 `12345678` 等弱密码；新版 APatch 管理器已强制校验强密码。

**方案 1：管理端修补 boot.img**——获取当前系统的 `boot.img`，设置超级密钥，使用 APatch 管理端修补后刷入。

**方案 2：第三方 Recovery 卡刷**——需要 TWRP、OrangeFox 等第三方 Recovery，使用高于版本 10888 的 APatch 安装包卡刷（下载后需将后缀改为 `.zip`）。通过这种方式刷入时，超级密钥为随机字母 + 数字组合。

**方案 3：Web 修补**——获取当前系统的 `boot.img`，设置超级密钥，使用 Kernel Patch on WEB 修补后刷入。

## Root 后的注意事项

建议使用 **Shamiko**、**TrickyStore** 等模块隐藏 Root 环境，减少被检测概率。

### Data 分区加密与 AVB 校验

出厂系统为 Android 10 及以上的手机默认启用 Data 加密。若手机性能较低，可能导致闪存访问速度下降；且在 Recovery 等环境下，若不解密 Data 分区，将无法访问 `/sdcard` 中的数据。

取消 Data 加密与 AVB 校验：需解包 `vendor` 分区，编辑其中的 `fstab.<平台>` 文件，删除对 `userdata` 分区的加密参数以及 AVB 校验参数，再重新打包刷回 `vendor` 分区。此操作风险较高，请仔细阅读机型对应的讨论帖后再尝试。

## 刷入新的系统

以 Pixel OS 为例：

1. 从官方网站下载 `rom.zip`、`recovery.img` 等文件。
2. 在 Fastboot 下刷入相应分区（如 `boot`、`recovery`、`vbmeta` 等）。
3. 重启进入 Recovery，执行：

```bat
adb sideload rom.zip
```

4. 清除手机数据后重启即可。

> 刷入前请先仔细阅读该 ROM 的 XDA 页面。部分 ROM 必须基于特定版本的底包刷入，否则轻则功能异常，重则直接变砖。类原生 UI 你可能在一周内看腻，且类原生系统没有厂商定制的各种服务，往往得不偿失。

## 变砖如何自救

### A. 更改重要分区导致 AVB 校验错误

重新刷入禁用 AVB 校验的 `vbmeta.img`：

```bat
fastboot --disable-verity --disable-verification flash vbmeta vbmeta.img
```

### B. 使用 KernelSU 因模块导致变砖

开机第一屏按**音量 −**：按下、松开、按下、松开、按下、松开，连续 3 次后松开按键，进入 KernelSU 内置的安全模式，卸载导致变砖的模块。

### C. 能进入 Fastboot 但不能进入 Fastbootd

进入 Fastboot，使用 **OPPO/OnePlus/Realme 官方线刷工具**解包当前机型对应系统版本的 `payload.bin`，在工具中选择「救砖」模式刷入。

### D. 不能进入 Fastboot 的手机

使用**深度刷机工具**进行救砖，或送售后。
