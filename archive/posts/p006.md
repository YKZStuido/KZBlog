## 为什么必须学指针

写 Python / JavaScript 的时候你从来不用关心「这个变量在哪」，反正能用就行。C++ 不一样——它让你**看得见、摸得着**内存。想把 C++ 写明白，指针是绕不过去的门槛。

好消息是：**指针本身不难**。它只是「一个变量，存的值是另一个变量的地址」。所有指针的复杂性，都是从这一句话延伸出来的。

## 变量在哪：地址

每个变量在内存里都有一个「**地址**」，可以理解为门牌号。C++ 里用 `&` 拿地址：

```cpp
#include <iostream>

int main() {
    int a = 42;
    std::cout << "a 的值 = " << a  << "\n";
    std::cout << "a 的地址 = " << &a << "\n";
    return 0;
}
```

输出类似：

```
a 的值 = 42
a 的地址 = 0x7ffee4a3c8ac
```

那串 `0x...` 就是 `a` 所在的内存地址（16 进制）。每次运行会变，不同机器也不一样。

::: callout `&` 的两种含义
- 出现在**类型**里（`int& r = a;`）→ **引用**。上一篇讲过。
- 出现在**表达式**里（`&a`）→ **取地址**。这里讲的是这个。

长得一样，意思不同。看位置区分。
:::

## 指针：装地址的变量

指针就是一个变量，它的值**是某个变量的地址**。类型写成「对应类型 + `*`」：

```cpp
int a = 42;
int* p = &a;       // p 是 int 指针，存着 a 的地址

std::cout << p  << "\n";   // 0x7ffee...   （地址）
std::cout << *p << "\n";   // 42            （通过指针读出 a 的值）
*p = 100;                  // 通过指针改 a
std::cout << a  << "\n";   // 100
```

拆开看：

- `int* p` 声明一个**指向 int 的指针**。
- `p = &a`：让 p 指向 a（装下 a 的地址）。
- `*p`（**解引用**）：顺着 p 找到那块内存，读/写里面的值。

**记住这张图**：

```
+---+           +---+
| p | --------> | a |
+---+  (地址)   +---+
                 42
```

### 指针类型为什么要区分？

`int*` 和 `double*` 都是「装地址的变量」，为什么不统一成一个 `pointer` 类型？因为**指针类型决定了「解引用时按什么类型读」**。

一个 `int*` 解引用读 4 字节（通常），一个 `double*` 读 8 字节。类型错了就读成垃圾。

### `*` 的两种含义

```cpp
int* p = &a;   // 声明里的 * → "这是指针类型"
*p = 100;      // 表达式里的 * → "解引用"
```

和 `&` 一样，位置决定含义。

## 空指针 `nullptr`

指针可以「什么都不指」。C++11 之后请用 `nullptr` 表示空指针：

```cpp
int* p = nullptr;

if (p) {
    std::cout << *p;   // 不会进来
} else {
    std::cout << "p 是空的\n";
}
```

::: callout 别用 `NULL`，也别用 `0`
老教材里会用 `NULL` 或者直接写 `0` 表示空指针。它们会和整数搞混，导致函数重载选错版本等怪事。**现代 C++ 一律 `nullptr`**。
:::

**对空指针解引用是未定义行为**——几乎肯定崩溃。用指针之前养成习惯先判断：

```cpp
if (p != nullptr) {
    // 可以安全 *p
}
```

## 栈 vs 堆

程序的内存主要分两块：**栈（stack）** 和 **堆（heap）**。

### 栈：函数局部变量的家

```cpp
void foo() {
    int x = 10;            // 在栈上
    int arr[100];          // 也在栈上
}   // 函数返回时，x 和 arr 自动消失
```

- **自动管理**：进函数时分配，出函数时回收，你不用管。
- **快**：分配 / 回收就是移动一个指针。
- **有限**：通常几 MB。开 `int arr[10000000]` 直接栈溢出。

### 堆：要多少有多少，要自己管

栈放不下的大东西，或者**生存期比当前函数长**的东西，放堆上。

- **手动分配**：用 `new`。
- **手动回收**：用 `delete`。
- **容量**：几 GB，看系统。
- **代价**：比栈慢一些，而且**忘了 `delete` 就是内存泄漏**。

## `new` / `delete`

```cpp
int* p = new int(42);       // 在堆上分配一个 int，初值 42
std::cout << *p << "\n";    // 42
delete p;                    // 释放
p = nullptr;                 // 好习惯：delete 之后置空
```

要分配数组，用 `new[]` 和 `delete[]`：

```cpp
int* arr = new int[100];
arr[0] = 1;
arr[99] = 99;
delete[] arr;                // 数组要用 delete[]，不能用 delete
arr = nullptr;
```

::: callout `new` 和 `delete` 必须配对
每个 `new` 都要有**恰好一次** `delete`。少了 → 内存泄漏。多了 → 重复释放，程序崩溃。`new[]` 必须配 `delete[]`，混用也是未定义行为。

**更重要的是**：现代 C++ 几乎不写裸 `new` / `delete`——用智能指针或容器代替。看下面的「现代 C++ 的答案」。
:::

### 为什么要在堆上分配？

一个常见场景：函数里造了个大东西，要**返回出去继续用**。

```cpp
int* make_array(int n) {
    int arr[1000];     // 栈上——函数结束就没了
    return arr;        // ← 危险：返回了个失效指针
}
```

上面这段是**典型错误**。改成堆上：

```cpp
int* make_array(int n) {
    int* arr = new int[n];    // 堆上——函数结束仍在
    return arr;                // 调用者负责 delete[]
}
```

当然，**更好的做法**是返回 `std::vector<int>`——自动管理内存，不用 `delete`：

```cpp
std::vector<int> make_vector(int n) {
    return std::vector<int>(n);
}
```

## 指针 vs 引用

都是"让函数能改外面的变量"，区别在哪？

| 特性 | 指针 `T*` | 引用 `T&` |
|---|---|---|
| 能不能为空 | 能（`nullptr`） | 不能，必须绑定到一个对象 |
| 能不能改指 | 能，随时指向别的 | 不能，一旦绑定就绑定到底 |
| 访问语法 | `*p` 或 `p->x` | 像普通变量 `r` / `r.x` |
| 取地址 | `&p` 是指针本身的地址 | `&r` 是被引用对象的地址 |

**选择建议：**

- 参数只用来**让函数改外面值** / 避免拷贝 → **引用**。
- 参数**可能为空**（比如「没有就传 null」的 API）→ **指针**。
- 参数要**在函数里换指向** → **指针**。
- 存在**动态分配**的对象 → 过去用指针，**现在用智能指针**。

## 指针和数组

C++ 里数组名在大多数表达式里会"衰变"成**指向第一个元素的指针**：

```cpp
int arr[5] = {10, 20, 30, 40, 50};
int* p = arr;           // 等价于 int* p = &arr[0];

std::cout << *p << "\n";        // 10
std::cout << *(p + 2) << "\n";  // 30
std::cout << p[2] << "\n";      // 30   ← 和上面等价！
```

`p[i]` 就是 `*(p + i)` 的语法糖——**数组下标访问其实是指针算术**。这是 C 的遗产，也是现代 C++ 推荐你**用 `std::vector` 和范围 for** 的原因：少一些踩坑机会。

::: callout C 字符串就是一个 char 指针
`"hello"` 这种字面量，类型是 `const char*`——一个指向以 `\0` 结尾的字符数组的指针。`std::string` 内部也是用一块堆内存加一个指针实现的，只是它帮你管好了。
:::

## 成员访问：`.` vs `->`

当你有一个**对象**时用 `.` 访问成员，有**指针**时用 `->`：

```cpp
std::string s = "hi";
s.size();          // 对象用 .

std::string* p = &s;
p->size();         // 指针用 ->
(*p).size();       // 等价写法，很少用
```

`p->size()` 就是 `(*p).size()` 的简写——先解引用再取成员。

## 现代 C++ 的答案：智能指针

手动 `new` / `delete` 太容易出错——忘记 delete、异常路径没 delete、重复 delete……工业界的答案是**智能指针**：**行为像指针，生命周期自己管**。

```cpp
#include <memory>

int main() {
    auto p = std::make_unique<int>(42);   // 堆上分配一个 int，自动管理
    std::cout << *p << "\n";               // 42
    // 函数结束时 p 析构，自动 delete——不用手写
}
```

两种最常用的：

- **`std::unique_ptr<T>`**：独占所有权。同一时刻只有一个 unique_ptr 指着那块内存。离开作用域就释放。
- **`std::shared_ptr<T>`**：共享所有权。多个 shared_ptr 指同一块内存，**最后一个走的时候**才释放（引用计数）。

::: callout 入门阶段的一条原则
**能不用裸指针就不用**。
- 存一组东西 → `std::vector`、`std::array`。
- 一个独占的动态对象 → `std::unique_ptr`。
- 只是"不想拷贝大对象" → `const T&`。

裸指针真正不可避免的场景（遍历链表、对接 C API、运行时多态的部分实现），后面学到了自然会用。**入门阶段别主动写 `new`**。
:::

## 一个小例子：链表节点

链表是指针最经典的用武之地——每个节点存下一个节点的地址。

```cpp
#include <iostream>

struct Node {
    int value;
    Node* next;
};

int main() {
    Node a{1, nullptr};
    Node b{2, nullptr};
    Node c{3, nullptr};

    a.next = &b;
    b.next = &c;

    // 从 a 开始遍历整个链表
    Node* cur = &a;
    while (cur != nullptr) {
        std::cout << cur->value << " ";
        cur = cur->next;
    }
    std::cout << "\n";   // 1 2 3
    return 0;
}
```

这里三个节点都在**栈**上，各自的 `next` 是另一个栈变量的地址，靠指针串起来。实战里通常会 `new` 在堆上，或者用 `std::unique_ptr` 自动管理——但思路是一样的。

## 常见坑

::: callout 1. 野指针（wild pointer）
```cpp
int* p;           // 没初始化，里面是随机地址
std::cout << *p;  // 崩
```
**指针声明后一定要赋值**——指向一个真实的变量，或者 `nullptr`。
:::

::: callout 2. 悬空指针（dangling pointer）
```cpp
int* p = new int(5);
delete p;
std::cout << *p;   // p 还指着那块地址，但那块已经不属于你了
```
**`delete` 之后把指针置成 `nullptr`**，至少再访问会崩得直接——比读到诡异的数据好得多。
:::

::: callout 3. 返回局部变量的地址
```cpp
int* bad() {
    int x = 42;
    return &x;     // x 在函数返回时就没了
}
```
和悬空指针本质一样——指针仍在，但指的对象已经消失。
:::

::: callout 4. `delete` 了栈上变量
```cpp
int a = 5;
int* p = &a;
delete p;          // 崩：a 不是 new 出来的
```
只有 `new` 出来的才能 `delete`。栈变量、全局变量、数组名、智能指针管的内存——都不能 `delete`。
:::

::: callout 5. 内存泄漏
```cpp
void f() {
    int* p = new int(5);
    if (some_condition) return;   // 忘了 delete
    delete p;
}
```
写着写着就忘了某条路径。**这正是智能指针要解决的问题**——它离开作用域自动释放，任你怎么 return 都不会漏。
:::
