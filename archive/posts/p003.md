## 为什么要先搭环境

C++ 和 Python、JavaScript 不一样，**不能直接运行源码**。你写的 `.cpp` 文件只是一段文本，必须经过**编译器**翻译成机器能认识的可执行文件（Windows 下是 `.exe`，Linux/macOS 下通常没有后缀），才能跑起来。

所以一套 C++ 环境至少包含两样东西：

- **编译器**：把源码翻译成可执行文件。主流选择是 **GCC**、**Clang** 和 **MSVC**。
- **编辑器 / IDE**：写代码的地方。轻量一点用 VS Code，重度一点用 Visual Studio 或 CLion。

> 编译器是**必须装**的，编辑器只要能写文本都行——但好的编辑器能帮你补全、查错、一键编译，强烈建议用 VS Code 起步。

## 选编译器

三大编译器的特点如下：

- **GCC（g++）**：Linux 默认编译器，Windows 上通过 MinGW-w64 / MSYS2 使用。社区最大，兼容性最好。
- **Clang**：macOS 默认编译器（装了 Xcode Command Line Tools 就自带），报错信息比 GCC 更人性化。
- **MSVC**：微软自家的 C++ 编译器，只在 Windows 上用，跟着 Visual Studio 一起装。

**新手推荐：**

- Windows → **MSYS2 + g++**
- macOS → **Clang**（自带）
- Linux → **g++**（发行版仓库里就有）

## Windows：用 MSYS2 装 g++

MSYS2 是 Windows 上最靠谱的 GCC 发行渠道，包管理器 `pacman` 还能顺带装 CMake、gdb 等工具。

1. 到 `https://www.msys2.org` 下载安装器，一路默认下一步。
2. 装完后从开始菜单打开 **MSYS2 UCRT64**（注意不是 MSYS 那个蓝色图标）。
3. 在终端里执行下面这行，装 g++ 和调试器：

```bash
pacman -S --needed mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-gdb
```

4. 把 `C:\msys64\ucrt64\bin` 加到系统的 `Path` 环境变量里（设置 → 系统 → 关于 → 高级系统设置 → 环境变量）。
5. 开一个**新的**命令提示符，验证：

```bat
g++ --version
```

看到类似 `g++ (Rev1, Built by MSYS2 project) 13.2.0` 就算成功。

::: callout 关于「MSVC 还是 MinGW」
如果你将来想写 Windows 原生 GUI 程序、用 DirectX，或者用 Visual Studio 调试大型工程，那就去下 **Visual Studio Community**（免费），它会把 MSVC 一起装好。本合集的例子用 g++ 和 MSVC 都能跑，不影响学习。
:::

## macOS：装 Xcode Command Line Tools

macOS 不用装 GCC，用自带的 Clang 就够了。打开「终端」，输入：

```bash
xcode-select --install
```

弹窗里点「安装」，等几分钟。装完验证：

```bash
clang++ --version
```

> macOS 上输入 `g++` 其实跑的也是 Clang，是苹果做的兼容别名。两者都可以用。

## Linux：一行命令搞定

以 Ubuntu / Debian 为例：

```bash
sudo apt update
sudo apt install build-essential gdb
```

`build-essential` 是个元包，会把 `g++`、`make` 等常用工具一起拉下来。其他发行版对应命令：

- Fedora：`sudo dnf install gcc-c++ gdb`
- Arch：`sudo pacman -S base-devel gdb`

验证：

```bash
g++ --version
```

## 选编辑器

**强烈推荐 VS Code** 作为起步编辑器——免费、跨平台、插件多、启动快。

1. 从 `https://code.visualstudio.com` 下载安装。
2. 打开 VS Code，左侧栏点「扩展」（或按 `Ctrl+Shift+X`）。
3. 搜索并安装 **C/C++**（Microsoft 出品，图标是蓝色的齿轮）。
4. 可选再装 **Code Runner**，能一键运行当前文件。

## 第一次编译运行

找个你喜欢的目录，新建文件 `hello.cpp`，粘贴下面的内容：

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, C++!" << std::endl;
    return 0;
}
```

在 VS Code 里按 `` Ctrl+` `` 打开终端（macOS 是 `` Cmd+` ``），执行：

```bash
g++ hello.cpp -o hello
./hello
```

Windows 下第二行是 `hello` 或 `.\hello`（不用写 `./`）。正常情况下你会看到：

```
Hello, C++!
```

恭喜——**你已经写出并运行了人生第一个 C++ 程序**。

### 拆解这条编译命令

- `g++`：调用 GCC 的 C++ 前端。
- `hello.cpp`：输入的源码文件。
- `-o hello`：把输出的可执行文件命名为 `hello`；**不写 `-o` 的话**，输出会叫 `a.out`（Linux/macOS）或 `a.exe`（Windows）。

> `-o` 是「output」的缩写。记住它——以后 99% 的编译命令都会用到。

## 推荐的日常参数

裸跑 `g++ hello.cpp` 能编译，但在学习阶段建议固定加上下面这组参数，**编译器会帮你发现大量隐藏 bug**：

```bash
g++ -std=c++17 -Wall -Wextra -O2 hello.cpp -o hello
```

- `-std=c++17`：使用 C++17 标准。现代教程基本都基于 C++17 或更高版本。
- `-Wall -Wextra`：打开常用警告。**警告就是金矿**，看懂它能少踩无数坑。
- `-O2`：开启优化，跑得更快（学习阶段影响不大，但习惯加上）。

嫌每次输入太长？写一个小脚本 `run.sh`（macOS/Linux）：

```bash
#!/bin/bash
g++ -std=c++17 -Wall -Wextra -O2 "$1" -o /tmp/a.out && /tmp/a.out
```

然后 `bash run.sh hello.cpp` 就行。

## 在线环境（临时救急）

嫌装环境麻烦，或者在别人的电脑上想写两行？试试这些：

- **Compiler Explorer**（`https://godbolt.org`）：能看到源码对应的汇编，学语言内部机制神器。
- **Wandbox**（`https://wandbox.org`）：多编译器版本切换方便。
- **Replit**（`https://replit.com`）：浏览器里就是一个完整 IDE。

不过**长期学习还是建议本地搭环境**——网速、保存、调试、文件管理都更顺手。

## 常见坑

::: callout 问题 1：`g++: command not found`
`PATH` 没配好。Windows 确认 `C:\msys64\ucrt64\bin` 加进了系统环境变量，**并且开了新终端**（老终端的环境变量不会自动更新）；macOS/Linux 确认装过对应软件包。
:::

::: callout 问题 2：中文输出乱码
Windows 默认终端编码是 GBK，源码里的中文字符串是 UTF-8。最简单的做法——执行 `chcp 65001` 切到 UTF-8，或者在 VS Code 终端里跑（VS Code 默认用 UTF-8）。
:::

::: callout 问题 3：`'iostream' file not found`
说明编译器本身没找到标准库。MSYS2 用户请确认用的是 **UCRT64** 那个终端，不是 MSYS 那个。macOS 如果报这个，跑 `xcode-select --install` 重装命令行工具。
:::
