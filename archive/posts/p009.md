## 为什么需要模板

写一个"两个数取最大值"的函数：

```cpp
int max_int(int a, int b) {
    return a > b ? a : b;
}
```

很快你就会想要 `double` 版、`std::string` 版、自己定义的 `Point` 版……每一种都要复制一遍代码，函数体一模一样，只是类型不同。

```cpp
double max_double(double a, double b) { return a > b ? a : b; }
std::string max_string(const std::string& a, const std::string& b) { return a > b ? a : b; }
// ...
```

C 的解法是宏（`#define MAX(a, b) ((a) > (b) ? (a) : (b))`），但宏没类型检查、容易踩坑。**C++ 的答案是模板**——一份代码，对任意类型都能用：

```cpp
template <typename T>
T max_value(T a, T b) {
    return a > b ? a : b;
}

max_value(3, 5);              // T = int
max_value(1.5, 2.5);          // T = double
max_value<std::string>("a", "b");   // T = std::string
```

编译器看到调用，**自动按用到的类型生成对应版本**——这个过程叫**实例化（instantiation）**。

::: callout 模板不是运行时多态
模板是**编译期**展开的——`max_value<int>` 和 `max_value<double>` 是编译器生成的两份独立代码，没有运行时开销。这和虚函数（运行时多态）是完全不同的两套机制。
:::

## 函数模板

### 基本写法

```cpp
template <typename T>
T add(T a, T b) {
    return a + b;
}
```

- `template <typename T>`：声明一个**类型参数 `T`**。
- `typename` 也可以写成 `class`，**意思一样**——历史包袱，挑一个用就行（现在主流写 `typename`）。
- 函数体里 `T` 当成"将来会被替换的类型"。

调用时既可以**显式指定**类型：

```cpp
add<int>(3, 5);
add<double>(1.5, 2.5);
```

也可以让编译器**根据参数推导**：

```cpp
add(3, 5);          // 推导 T = int
add(1.5, 2.5);      // 推导 T = double
add(3, 5.0);        // 编译错：3 是 int，5.0 是 double，不知道 T 是哪个
```

最后一行的歧义可以靠显式指定解决：`add<double>(3, 5.0)`。

### 多个类型参数

```cpp
template <typename T, typename U>
auto multiply(T a, U b) {
    return a * b;
}

multiply(3, 1.5);   // T=int, U=double, 返回 double
```

返回类型用 `auto` 让编译器推——`T * U` 的结果类型可能既不是 `T` 也不是 `U`（`int * double = double`）。

### 模板实参的"鸭子类型"

模板对类型只有一个隐含要求：**用到的操作得支持**。

```cpp
template <typename T>
T max_value(T a, T b) {
    return a > b ? a : b;     // 要求 T 支持 >
}

struct Foo {};
max_value(Foo{}, Foo{});       // 编译错：Foo 没有 operator>
```

报错信息会指向函数体里的那个 `>`，新手常被吓到——**模板出错时报错信息特别长**，但本质就是"你给的类型不支持某个操作"。看错误的最后几行，找到 `note: in instantiation of` 那行就行。

## 类模板

容器、智能指针、几乎一切"对类型通用"的类都是类模板。最简单的例子：写一个自己的 `Pair`：

```cpp
template <typename T1, typename T2>
class Pair {
public:
    Pair(T1 first, T2 second) : first_(first), second_(second) {}

    T1 first() const { return first_; }
    T2 second() const { return second_; }

private:
    T1 first_;
    T2 second_;
};

Pair<int, std::string> p(1, "abc");
std::cout << p.first() << " " << p.second() << "\n";
```

注意几个细节：

- 实例化时**必须显式指定类型**：`Pair<int, std::string>`，类模板不像函数模板那样自动推导（C++17 之后类模板也能推导，先不展开）。
- 成员函数定义在类外的话，**每个函数前都要重复写 `template`**：

```cpp
template <typename T1, typename T2>
T1 Pair<T1, T2>::first() const {
    return first_;
}
```

挺啰嗦，但只要你按上面那样写在类内，就不用管这个细节。

### 一个能用的小例子：自己的 `Stack<T>`

```cpp
#include <vector>
#include <stdexcept>

template <typename T>
class Stack {
public:
    void push(const T& value) { data_.push_back(value); }

    void pop() {
        if (data_.empty()) throw std::out_of_range("stack is empty");
        data_.pop_back();
    }

    T& top() {
        if (data_.empty()) throw std::out_of_range("stack is empty");
        return data_.back();
    }

    bool empty() const { return data_.empty(); }
    size_t size() const { return data_.size(); }

private:
    std::vector<T> data_;
};

int main() {
    Stack<int> s;
    s.push(1);
    s.push(2);
    s.push(3);
    while (!s.empty()) {
        std::cout << s.top() << " ";
        s.pop();
    }
    // 3 2 1
}
```

这就是 STL 容器的基本思路——把"装什么类型"交给用户，自己只管"怎么装"。

## 模板参数不一定是类型

除了 `typename T`，模板参数还能是**整数等编译期常量**：

```cpp
template <typename T, size_t N>
class FixedArray {
public:
    T& operator[](size_t i) { return data_[i]; }
    size_t size() const { return N; }
private:
    T data_[N];
};

FixedArray<int, 5> arr;
arr[0] = 42;
std::cout << arr.size() << "\n";   // 5
```

这就是 `std::array<T, N>` 的写法。`N` 必须是**编译期能确定的常量**——不能是变量。

## `auto` 和模板的关系

`auto` 在很多地方就是模板推导的别名：

```cpp
auto x = 42;             // x 是 int

template <typename T>
void f(T x);
f(42);                   // T = int
```

C++14 还支持**泛型 lambda**——参数写 `auto` 等于隐式模板：

```cpp
auto add = [](auto a, auto b) {
    return a + b;
};
add(1, 2);          // int
add(1.5, 2.5);      // double
add(std::string("a"), std::string("b"));   // string
```

写一些小工具特别顺手。

## 模板特化：对特定类型走专门的实现

有时通用版本对某个类型不合适，要"为这个类型单独写一份"——这叫**特化（specialization）**。

```cpp
template <typename T>
struct TypeName {
    static const char* name() { return "unknown"; }
};

// 对 int 特化
template <>
struct TypeName<int> {
    static const char* name() { return "int"; }
};

// 对 double 特化
template <>
struct TypeName<double> {
    static const char* name() { return "double"; }
};

std::cout << TypeName<int>::name() << "\n";       // int
std::cout << TypeName<double>::name() << "\n";    // double
std::cout << TypeName<char>::name() << "\n";      // unknown（用通用版）
```

入门阶段**知道有这回事就行**——你写自己的代码很少需要特化，但读 STL / 第三方库源码时经常会看到。

## 可变参数模板：参数个数也能变

C++11 引入了**可变参数模板（variadic template）**——参数个数不固定：

```cpp
#include <iostream>

void print() {
    std::cout << "\n";          // 0 个参数时的版本
}

template <typename T, typename... Rest>
void print(T first, Rest... rest) {
    std::cout << first << " ";
    print(rest...);              // 把剩下的递归传下去
}

int main() {
    print(1, 2.5, "hello", 'c');
    // 1 2.5 hello c
}
```

`typename... Rest` 表示"零个或多个类型参数"，`rest...` 表示"把它们展开"。这是 `printf` 那种"传任意多参数"的现代、类型安全版。

C++17 之后可以用**折叠表达式（fold expression）**写得更短：

```cpp
template <typename... Args>
void print(Args... args) {
    ((std::cout << args << " "), ...);   // 折叠：对每个 arg 执行
    std::cout << "\n";
}
```

入门阶段不用现在就掌握——但你**一定会**在读现代 C++ 库时碰到 `...`，知道它的意思就够。

## C++20 的 concepts：给模板加约束

普通模板的"鸭子类型"很自由，但**报错难看**且**很难表达"我只接受能比较的类型"**。C++20 引入 **concepts** 解决这个：

```cpp
#include <concepts>

template <typename T>
    requires std::totally_ordered<T>     // 要求 T 支持完整比较
T max_value(T a, T b) {
    return a > b ? a : b;
}

max_value(3, 5);                         // 没问题
struct Foo {};
max_value(Foo{}, Foo{});                 // 编译错：Foo 不满足 totally_ordered
                                          // 报错信息直接说"不满足约束"
```

更短的写法是直接用 concept 替换 `typename`：

```cpp
template <std::totally_ordered T>
T max_value(T a, T b) {
    return a > b ? a : b;
}
```

::: callout 编译器要够新
concepts 要 GCC 10+ / Clang 13+ / MSVC 19.30+，编译时还得加 `-std=c++20`。如果你跟着这个合集学，**前几次见到 concepts 不用慌**——能不用就先不用，等后面用顺了再升级。
:::

## 模板放头文件的"奇怪规则"

普通函数 `.h` 声明、`.cpp` 定义。**模板不能这么干**——模板的定义**必须**对每个使用它的 `.cpp` 可见。所以：

```cpp
// stack.h
template <typename T>
class Stack {
public:
    void push(const T& v);
    // ...
};

template <typename T>           // 定义也写在头文件里
void Stack<T>::push(const T& v) { /* ... */ }
```

或者干脆**全部写在类内**（最简单）。如果你把模板定义放到 `.cpp` 里，链接时会报"undefined reference"——这是模板入门最典型的踩坑点之一。

::: callout 为什么模板必须在头文件
编译器实例化 `Stack<int>` 时需要看到完整定义。如果定义在另一个 `.cpp` 里，**当前编译单元根本不知道怎么生成 `Stack<int>`**——链接阶段才发现，但那时编译已经结束了。
:::

## 一个综合例子：通用的 `find_if`

写一个简化版的 `find_if`——在容器里找第一个满足条件的元素：

```cpp
#include <iostream>
#include <vector>
#include <list>
#include <string>

template <typename Container, typename Predicate>
auto find_if_my(const Container& c, Predicate pred) {
    for (auto it = c.begin(); it != c.end(); ++it) {
        if (pred(*it)) return it;
    }
    return c.end();
}

int main() {
    std::vector<int> v{1, 3, 5, 8, 9};
    auto it1 = find_if_my(v, [](int x) { return x % 2 == 0; });
    if (it1 != v.end()) std::cout << "第一个偶数：" << *it1 << "\n";   // 8

    std::list<std::string> names{"Alice", "Bob", "Charlie"};
    auto it2 = find_if_my(names, [](const std::string& s) {
        return s.size() > 3;
    });
    if (it2 != names.end()) std::cout << "第一个长于 3 的：" << *it2 << "\n";   // Alice

    return 0;
}
```

`Container` 任意容器、`Predicate` 任意可调用对象——一份代码全包了。这就是模板的威力：**写一次，到处用**。

## 常见坑

::: callout 1. 模板代码必须可见
把模板定义放进 `.cpp` 文件，链接报 `undefined reference`。**模板的实现必须放在头文件里**，要么写在类内，要么在类外用 `template <...>` 重复声明。
:::

::: callout 2. 错误信息又长又恐怖
模板出错动辄几百行报错——编译器把整条实例化链都吐给你。**看最后几行**，找 `note: in instantiation of ...` 那行，往上找原始的 `error:`，那才是真正的错误。
:::

::: callout 3. 类模板不会自动推导（C++17 前）
```cpp
Pair p(1, 2.5);                  // C++17 之前编译错
Pair<int, double> p(1, 2.5);     // 必须显式
```
C++17 后引入了 CTAD（class template argument deduction），但很多老代码、老编译器不支持。**写类模板时显式指定参数最稳妥**。
:::

::: callout 4. `typename` 在依赖类型里是必须的
```cpp
template <typename T>
void f() {
    T::value_type x;             // 编译错
    typename T::value_type x;    // 对
}
```
`T::value_type` 在编译器看来可能是类型、也可能是值（编译器没办法预先知道）。**确认它是类型时必须加 `typename`**。
:::

::: callout 5. 模板的"过度通用化"
```cpp
template <typename T>
T add(T a, T b) { return a + b; }
```
够通用了？接 `int*` 进去也能编译——`int* + int*` 是非法的，但报错点会指向**调用处而不是定义处**。**写模板时心里要有一个清单**：T 需要支持哪些操作？理想情况下 C++20 写成 concept，至少在文档里说清楚。
:::
