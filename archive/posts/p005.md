## 为什么要把函数单独拎出来讲

上一篇把函数的「长什么样」讲过了——返回类型、函数名、参数列表、函数体。但写真正的程序时，你会反复撞到这几个问题：

- 参数到底是**拷贝一份**给函数，还是**让函数改我外面的变量**？
- 两个函数能不能**同名**？编译器怎么知道调哪个？
- 参数能不能**有默认值**？
- 递归怎么才不会栈溢出？
- 听别人说的 `[](){}` 是什么？

一口气过一遍。

## 参数传递：三种方式

C++ 传参有三种路子。现在理解清楚，后面读别人代码就不会懵。

### 传值（pass by value）

最直白——函数拿到的是一份**独立拷贝**，改它不影响外面。

```cpp
void bump(int x) {
    x += 1;
}

int a = 5;
bump(a);
std::cout << a << "\n";   // 依然是 5
```

函数签名 `void bump(int x)` 里没有 `&` 也没有 `*`，就是传值。

**优点**：安全，互不干扰。
**缺点**：拷贝有成本。传一个 `int` 无所谓，传一个几 MB 的 `std::vector` 就要命了。

### 传引用（pass by reference）

加一个 `&`，函数拿到的是**别名**——改它就是改外面的原变量。

```cpp
void bump(int& x) {
    x += 1;
}

int a = 5;
bump(a);
std::cout << a << "\n";   // 6
```

什么时候用引用？

- **要让函数修改外面的值**（比如经典的 `swap`）。
- **传大对象但不想拷贝**。

```cpp
void swap_ints(int& a, int& b) {
    int tmp = a;
    a = b;
    b = tmp;
}
```

### `const` 引用：不拷贝也不让改

传大对象又不想让函数改，就是 `const` 引用——**读得到，改不了，不拷贝**。这是 C++ 里最常用的参数形式。

```cpp
void print(const std::string& s) {
    std::cout << s << "\n";
    // s = "hi";   // 编译错：不能改 const 引用
}
```

::: callout 一条黄金规则
- **基础类型**（`int`、`double`、`char`、`bool`）→ 传值。
- **大对象**（`std::string`、`std::vector`、自定义类）→ **`const T&`**（只读）或 **`T&`**（要改）。
:::

### 传指针呢？

C++ 里也能传指针，写成 `void bump(int* x) { *x += 1; }`，调用时写 `bump(&a);`。能做的事和引用差不多，但指针可以为空（`nullptr`）、可以重新指向别的东西。**下一篇**我们专门讲指针，这里先不展开。

## 返回值

返回值和参数一样，也有三种——按值、按引用、按指针。

按值最简单，也是新手默认写的：

```cpp
std::string greet(const std::string& name) {
    return "Hello, " + name;
}
```

> 别担心「返回一个字符串不是要拷贝吗」。现代编译器有 **RVO**（返回值优化），这种写法在绝大多数情况下不会产生多余拷贝，放心写。

**不要返回局部变量的引用**——这是新手 top 5 级别的坑：

```cpp
const std::string& bad() {
    std::string s = "hi";
    return s;         // s 在函数结束时就没了
}                      // 返回的引用指向一块「已经失效」的内存
```

拿到这个引用再去读，程序不是崩就是拿到垃圾。**返回引用只在「返回的是外面本来就存在的东西」时才合适**（比如类的成员函数返回成员变量的引用）。

## 默认参数

参数可以有默认值，调用时不写就用默认值：

```cpp
void log(const std::string& msg, const std::string& level = "INFO") {
    std::cout << "[" << level << "] " << msg << "\n";
}

log("开机");                 // [INFO] 开机
log("磁盘快满了", "WARN");    // [WARN] 磁盘快满了
```

两条规矩要记住：

1. **默认参数必须放在参数列表的末尾**。不能 `void f(int a = 1, int b)`，只能 `void f(int a, int b = 1)`。
2. **默认参数写在声明里**，别在定义里重复写：

```cpp
// .h
void log(const std::string& msg, const std::string& level = "INFO");

// .cpp
void log(const std::string& msg, const std::string& level) {
    // ...
}
```

## 函数重载：同名不同签名

C++ 允许多个函数**同名**，只要**参数列表不同**，编译器能区分：

```cpp
int add(int a, int b)       { return a + b; }
double add(double a, double b) { return a + b; }
std::string add(const std::string& a, const std::string& b) {
    return a + b;
}

add(1, 2);              // int 版本
add(1.5, 2.5);          // double 版本
add(std::string("ya"), std::string("ke"));   // string 版本
```

::: callout 重载靠的是「参数」，不是「返回值」
```cpp
int  f(int);
double f(int);     // 编译错：光靠返回值区分不了
```
调用 `f(3)` 时编译器不知道你想要哪个，所以 C++ 直接禁止这种写法。
:::

重载在**写库**时特别常见——比如 `std::to_string`，对 `int`、`double`、`long` 等都有一个版本，用起来无感。

## 递归：函数调用自己

一个函数可以在自己的函数体里调用自己。经典例子是阶乘：

```cpp
int factorial(int n) {
    if (n <= 1) return 1;         // 基线条件
    return n * factorial(n - 1);  // 递归调用
}

std::cout << factorial(5) << "\n";   // 120
```

写递归的思路永远是两步：

1. **基线条件（base case）**：最小的那个情况直接返回结果。**先写这个**。
2. **递归条件**：用更小的问题的结果，拼出当前问题的结果。

`factorial(5)` 实际发生的事：

```
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 5 * 4 * 3 * 2 * 1
= 120
```

练习题目：[B2142 求 1+2+3+...+N 的值](https://www.luogu.com.cn/problem/B2142)

### 斐波那契：一个反面教材

```cpp
int fib(int n) {
    if (n < 2) return n;
    return fib(n - 1) + fib(n - 2);
}
```

写法优雅，但 `fib(40)` 会慢到怀疑人生——因为重复算了很多遍同样的子问题。这种情况要改成**循环 + 记忆**：

```cpp
int fib(int n) {
    if (n < 2) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; ++i) {
        int c = a + b;
        a = b;
        b = c;
    }
    return b;
}
```

::: callout 递归 vs 循环
能写成循环的，优先用循环——更快、更省栈。递归的真正舞台是**结构本身递归的问题**：树的遍历、分治算法、回溯搜索。阶乘和斐波那契只是教学例子，实战里没人那么写。
:::

### 栈溢出

每调一次函数，都会在**调用栈**上占一块空间。递归太深会把栈用光，程序崩溃（Windows 上报 `stack overflow`，Linux 上 `Segmentation fault`）。

一个空函数大约能递归几万层；实际有局部变量的函数，能递归几千到几万层。**如果你的递归深度和输入规模直接挂钩，就要小心了**。

## 内联与 `inline`

看到别人在函数前写 `inline`？

```cpp
inline int square(int x) { return x * x; }
```

`inline` 是一个**建议**，告诉编译器「这个函数你可以把它的代码直接插到调用点，而不是真的调用」。小函数这样做能少一次跳转，可能更快。**但现在编译器比你聪明**——它自己会判断。

`inline` 在现代 C++ 里主要的用途其实是**允许一个函数在多个 `.cpp` 里重复定义而不链接报错**（头文件里定义函数时必须加 `inline`）。

入门阶段：**不用主动加 `inline`**，知道看到它不慌就行。

## Lambda 表达式

写完一个函数，就为了传给另一个函数用一次，挺亏的。**Lambda** 是「就地定义的小函数」：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6};

    // 把 v 按从大到小排序
    std::sort(v.begin(), v.end(), [](int a, int b) {
        return a > b;
    });

    for (int x : v) std::cout << x << " ";
    std::cout << "\n";
}
```

`[](int a, int b) { return a > b; }` 就是一个 lambda。拆开看：

- `[]`：**捕获列表**，里面写「这个 lambda 要不要带走外面作用域的变量」。空的表示不带。
- `(int a, int b)`：参数列表，和普通函数一样。
- `{ return a > b; }`：函数体。
- 返回类型由编译器自动推导（和 `auto` 一样的思路）。

### 捕获外面的变量

```cpp
int threshold = 10;

auto is_big = [threshold](int x) {
    return x > threshold;
};

std::cout << is_big(15) << "\n";   // 1
std::cout << is_big(5)  << "\n";   // 0
```

`[threshold]` 把外面的 `threshold` **按值**带进 lambda；想按引用带就写 `[&threshold]`。`[=]` 和 `[&]` 表示「把能用到的所有外部变量都按值 / 按引用带进来」，能用但**不推荐**——写明确更清楚。

入门阶段只需要记住：**需要把一小段逻辑传给 STL 算法时，用 lambda 最顺手**。

## 一个综合小例子

把「筛选 + 排序 + 求和」串起来——过滤出 vector 里大于某阈值的数，从大到小排序，打印，顺带求和。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int sum_above(std::vector<int> v, int threshold) {
    // 按大到小排序
    std::sort(v.begin(), v.end(), [](int a, int b) { return a > b; });

    int total = 0;
    std::cout << "大于 " << threshold << " 的：";
    for (int x : v) {
        if (x <= threshold) break;   // 已经降序，一旦不满足就停
        std::cout << x << " ";
        total += x;
    }
    std::cout << "\n";
    return total;
}

int main() {
    std::vector<int> nums = {3, 14, 1, 5, 9, 26, 53, 5, 8};
    int s = sum_above(nums, 10);
    std::cout << "和 = " << s << "\n";
    return 0;
}
```

这段代码你应该能一眼看懂每一部分——这就是这一篇的目标。

## 常见坑

::: callout 1. 传了个大对象没加引用
`void print(std::string s)` 每次调用都要拷贝字符串。改成 `const std::string&`，既只读又不拷贝。
:::

::: callout 2. 把引用当指针用
引用一旦绑定到一个变量，就**不能改指**到别的变量。`int& r = a; r = b;` 不是让 r 指向 b——是把 b 的值赋给 a。需要「能重新指」的语义时用指针。
:::

::: callout 3. 忘记 `return`
不是 `void` 的函数一定要返回一个值。没写 return 编译器通常会警告（`-Wall` 打开的话），但运行时行为是未定义的——什么值都可能。
:::

::: callout 4. 递归忘了基线条件
`int f(int n) { return f(n - 1); }` 永远不会返回，程序直接栈溢出。**写递归第一件事是写终止条件**。
:::

::: callout 5. 默认参数写在定义里
```cpp
// .h: void f(int x);
// .cpp:
void f(int x = 10) { ... }   // 不行
```
默认参数应该写在头文件的**声明**里。入门阶段都写单文件的话感知不到这个区别，但早知道早踩不了坑。
:::
