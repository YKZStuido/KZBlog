## 为什么需要类

到目前为止你写的程序，是「一堆数据 + 一堆函数」分开管的。比如要描述一个学生：

```cpp
std::string name;
int age;
double score;

void print_student(const std::string& n, int a, double s) {
    std::cout << n << " " << a << " " << s << "\n";
}
```

数据散在各个变量里，函数挂在外面。学生数量一多，谁是谁的就乱了——`name1` 配 `age2`，bug 半小时找不到。

**类（class）** 就是把「相关的数据」和「操作这些数据的函数」**打包成一个新类型**。

```cpp
struct Student {
    std::string name;
    int age;
    double score;

    void print() const {
        std::cout << name << " " << age << " " << score << "\n";
    }
};

Student s{"亚柯洲", 13, 95.5};
s.print();
```

这就是面向对象最基本的形态——数据和操作捆在一起，互相找得到，调用方便。

::: callout `struct` 还是 `class`？
两者**几乎完全一样**，唯一的差别是默认访问权限：`struct` 默认 `public`，`class` 默认 `private`。

约定俗成：纯数据聚合用 `struct`，有行为和封装的复杂类型用 `class`。本篇为了少打几个 `public:` 大量用 `struct`，但你看到的真实工程代码里 `class` 更多。
:::

## 类的基本结构

```cpp
class Counter {
public:
    Counter() : count_(0) {}            // 构造函数
    ~Counter() {}                        // 析构函数

    void inc() { ++count_; }
    int get() const { return count_; }

private:
    int count_;                          // 成员变量（约定带下划线后缀）
};

int main() {
    Counter c;
    c.inc();
    c.inc();
    c.inc();
    std::cout << c.get() << "\n";        // 3
}
```

拆开看几个新东西：

- `public:` / `private:`：访问控制——`public` 的成员外面能用，`private` 的只有类自己能用。
- `Counter()`：**构造函数**，类被创建时自动调用。函数名和类名一样，没有返回类型。
- `~Counter()`：**析构函数**，对象消失时自动调用。函数名是类名前面加个 `~`。
- `count_`：成员变量，约定加个下划线后缀（或前缀），方便和参数、局部变量区分。
- `int get() const`：成员函数后面那个 `const` 表示「这个函数不会修改成员变量」。

### 构造函数：怎么"造"一个对象

构造函数最常见的写法是用**成员初始化列表**：

```cpp
class Point {
public:
    Point(double x, double y) : x_(x), y_(y) {}

    double distance() const {
        return std::sqrt(x_ * x_ + y_ * y_);
    }

private:
    double x_, y_;
};

Point p(3.0, 4.0);
std::cout << p.distance() << "\n";   // 5
```

`: x_(x), y_(y)` 这一段就是初始化列表，意思是「用参数 `x` 初始化成员 `x_`，用 `y` 初始化 `y_`」。

::: callout 为什么不写在函数体里？
你也能写成：

```cpp
Point(double x, double y) {
    x_ = x;
    y_ = y;
}
```

效果看起来一样，但**初始化列表是先分配再赋值**的最佳形式——对于内置类型差别不大，但成员是一个复杂类（比如 `std::string`）时，初始化列表少一次"先默认构造再赋值"的浪费。**养成用初始化列表的习惯**。
:::

### 重载构造函数

类也能有多个构造函数，参数不同就行：

```cpp
class Point {
public:
    Point() : x_(0), y_(0) {}                       // 默认构造
    Point(double x, double y) : x_(x), y_(y) {}     // 双参数
    Point(double v) : x_(v), y_(v) {}               // 单参数

private:
    double x_, y_;
};

Point p1;             // 0, 0
Point p2(3.0, 4.0);   // 3, 4
Point p3(5.0);        // 5, 5
```

C++11 之后还可以写**默认成员初始化**，把默认值写在声明里：

```cpp
class Point {
public:
    Point() = default;                   // 让编译器自动生成默认构造
    Point(double x, double y) : x_(x), y_(y) {}

private:
    double x_ = 0;
    double y_ = 0;
};
```

`= default` 告诉编译器："默认构造函数你自己写，规则就是默认的那一套"。简洁很多。

### 析构函数：消失时做什么

类离开作用域、被 `delete`、或者所在容器被销毁时，析构函数会自动调用。**它的工作是清理这个对象占着的资源**——比如关闭文件、释放堆内存、断开网络连接。

```cpp
class FileHandle {
public:
    FileHandle(const std::string& path) {
        fp_ = std::fopen(path.c_str(), "r");
    }
    ~FileHandle() {
        if (fp_) std::fclose(fp_);     // 自动收尾
    }

private:
    std::FILE* fp_;
};
```

只要把 `FileHandle` 当变量一用，离开作用域它就会**自动关文件**——不用你手动写 `fclose`。这就是 C++ 的核心思想 **RAII**（Resource Acquisition Is Initialization）：**资源的生命周期 = 对象的生命周期**。

::: callout 智能指针就是 RAII
上一篇讲过的 `std::unique_ptr`、`std::vector` 都是 RAII 的化身——构造时申请资源，析构时自动释放。**你写的类也应该这么做**：让用户 `new` 一下，剩下的我帮你管。
:::

## 封装：`private` 不是偏执，是甲方需求

为什么要把成员藏起来？看个例子：

```cpp
struct Account {
    double balance;
};

Account a{100.0};
a.balance = -99999.0;   // 余额为负？？？
```

成员是 `public`，外面能随便改，业务规则没法保证。改成封装版：

```cpp
class Account {
public:
    Account(double initial) : balance_(initial) {}

    void deposit(double amount) {
        if (amount > 0) balance_ += amount;
    }

    bool withdraw(double amount) {
        if (amount <= 0 || amount > balance_) return false;
        balance_ -= amount;
        return true;
    }

    double balance() const { return balance_; }

private:
    double balance_;
};
```

外面只能调 `deposit` / `withdraw`，没法直接动 `balance_`——业务规则**强制执行**。这就是封装：**让数据只通过受控的接口被修改**。

读者只看 `public` 接口就知道这个类能做什么；类的实现者随时可以改 `private` 的内部结构（比如把 `double` 换成定点数），不影响外面。

## `this` 指针

每个成员函数里都有一个隐式参数 **`this`**——指向"调用这个函数的那个对象"。

```cpp
class Box {
public:
    void resize(int width) {
        // 下面两个写法等价
        width_ = width;
        this->width_ = width;
    }

    Box& set_height(int h) {
        height_ = h;
        return *this;        // 返回自己，支持链式调用
    }

private:
    int width_, height_;
};

Box b;
b.set_height(10).set_height(20);   // 链式调用
```

99% 的情况下你不需要显式写 `this->`——直接用成员名就行。`this` 真正用得到是**返回 `*this` 实现链式调用**，或者**消除歧义**（参数名和成员名相同时）。

## 继承：复用与"is-a"关系

子类**继承**父类的所有成员，并能**添加或覆盖**自己的：

```cpp
class Animal {
public:
    Animal(const std::string& name) : name_(name) {}

    void eat() {
        std::cout << name_ << " 在吃东西\n";
    }

    void speak() const {
        std::cout << "..." << "\n";
    }

protected:
    std::string name_;     // 子类能访问，外面不行
};

class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) {}

    void speak() const {
        std::cout << name_ << "：汪汪！\n";
    }
};

Dog d("旺财");
d.eat();      // 旺财 在吃东西     ← 继承自 Animal
d.speak();    // 旺财：汪汪！      ← Dog 自己的
```

几个新词：

- `class Dog : public Animal`：表示 `Dog` **公有继承**自 `Animal`。
- `protected`：第三种访问权限，介于 `public` 和 `private` 之间——**子类能访问，外面不能**。
- `Dog(...) : Animal(name) {}`：子类构造函数里调用父类构造函数，**这是必须做的**——父类不会自动构造。

::: callout 别滥用继承
新手看完 OOP 教程容易写出 `Dog → Animal → LivingThing → PhysicalObject` 这种四五层继承的代码——**别这么干**。

工程里继承应该非常克制。**优先用组合**（一个类里持有另一个类的对象），只有在确实是「is-a」关系且需要**多态**时才用继承。Java 教科书里的"动物-狗-猫"在真实 C++ 代码里出现得不多。
:::

## 多态：父类指针 → 子类行为

继承真正的威力在**多态**——父类指针/引用能调用子类的实现。这需要一个关键字：**`virtual`**。

```cpp
class Animal {
public:
    Animal(const std::string& name) : name_(name) {}
    virtual ~Animal() = default;          // 重要！见下面的 callout

    virtual void speak() const {           // virtual：可以被子类覆盖
        std::cout << "..." << "\n";
    }

protected:
    std::string name_;
};

class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) {}

    void speak() const override {          // override：明确覆盖
        std::cout << name_ << "：汪汪！\n";
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& name) : Animal(name) {}

    void speak() const override {
        std::cout << name_ << "：喵～\n";
    }
};

void make_speak(const Animal& a) {
    a.speak();          // 真正调的是哪个？
}

int main() {
    Dog d("旺财");
    Cat c("咪咪");
    make_speak(d);      // 旺财：汪汪！
    make_speak(c);      // 咪咪：喵～
}
```

`make_speak` 拿到的是 `const Animal&`，但调用 `a.speak()` 时，**实际执行的是真实对象类型对应的版本**——这就是**运行时多态**。

### 几个关键词

- **`virtual`**：在父类里把这个函数标记为"允许被覆盖"。**没标 `virtual`，子类的同名函数不会被多态调用**——会变成"隐藏"，结果通常不是你想要的。
- **`override`**：在子类里明确说"我在覆盖父类的虚函数"。**强烈建议每次都加**——拼错函数名时编译器会立刻报错。
- **`= 0`**：把虚函数声明为**纯虚函数**——这个父类是"抽象类"，不能实例化，子类必须实现：

```cpp
class Shape {
public:
    virtual ~Shape() = default;
    virtual double area() const = 0;     // 纯虚
};

// Shape s;          // 编译错：抽象类不能实例化

class Circle : public Shape {
public:
    Circle(double r) : r_(r) {}
    double area() const override {
        return 3.14159265 * r_ * r_;
    }
private:
    double r_;
};
```

::: callout 父类析构函数必须 virtual
有多态的父类，**析构函数一定要写成 `virtual`**：

```cpp
Animal* a = new Dog("旺财");
delete a;       // 没 virtual 析构会出 bug：只析构了 Animal，没析构 Dog
```

不加 `virtual`，`delete` 时只调用 `Animal` 的析构函数，`Dog` 那部分资源就漏了。**只要类里有任何 virtual 函数，析构函数就要 virtual**。
:::

## 拷贝、移动与「三/五法则」

```cpp
Student a{"亚柯洲", 13, 95.5};
Student b = a;         // 拷贝构造
Student c;
c = a;                 // 拷贝赋值
```

C++ 默认会帮你生成拷贝构造、拷贝赋值、析构等函数。**对于只含基础类型和 STL 类型的类，默认生成的就够用**——不用管。

只有当类**自己管裸资源**（比如 `new` 出来的指针、文件句柄），默认行为才不对——会出现"两个对象指向同一块内存，析构两次"的灾难。这种情况你需要自己实现：拷贝构造、拷贝赋值、析构（**三法则**），C++11 之后加上移动构造、移动赋值（**五法则**）。

入门阶段只要记住一句：**优先用 STL 容器和智能指针，让默认行为正确**。需要自己管资源时再来翻五法则的书。

## 一个综合例子

实现一个简单的**形状系统**——把上面学的 OOP 串起来：

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Shape {
public:
    virtual ~Shape() = default;
    virtual double area() const = 0;
    virtual std::string name() const = 0;

    void describe() const {
        std::cout << name() << " 的面积 = " << area() << "\n";
    }
};

class Circle : public Shape {
public:
    Circle(double r) : r_(r) {}
    double area() const override { return 3.14159265 * r_ * r_; }
    std::string name() const override { return "圆"; }
private:
    double r_;
};

class Rectangle : public Shape {
public:
    Rectangle(double w, double h) : w_(w), h_(h) {}
    double area() const override { return w_ * h_; }
    std::string name() const override { return "矩形"; }
private:
    double w_, h_;
};

class Triangle : public Shape {
public:
    Triangle(double base, double h) : base_(base), h_(h) {}
    double area() const override { return 0.5 * base_ * h_; }
    std::string name() const override { return "三角形"; }
private:
    double base_, h_;
};

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(2.0));
    shapes.push_back(std::make_unique<Rectangle>(3.0, 4.0));
    shapes.push_back(std::make_unique<Triangle>(5.0, 6.0));

    double total = 0;
    for (const auto& s : shapes) {
        s->describe();
        total += s->area();
    }
    std::cout << "总面积 = " << total << "\n";
    return 0;
}
```

短短几十行覆盖了：纯虚函数 / 抽象类、`override`、`virtual` 析构、`std::unique_ptr` 持有多态对象、`std::vector` 装智能指针、范围 for + `auto`。**把这段代码亲手敲一遍**，下面再加一个 `Square` 试试。

## 常见坑

::: callout 1. 忘记父类构造函数
```cpp
class Dog : public Animal {
public:
    Dog(const std::string& name) {       // 没在初始化列表里调 Animal()
        // ...
    }
};
```
父类有非默认构造时，**子类必须**在初始化列表里显式调用：`: Animal(name)`。否则编译错或者父类被默认构造（如果存在）。
:::

::: callout 2. 父类析构忘加 virtual
有多态时父类析构必须 `virtual`，否则 `delete` 父类指针会泄漏子类资源。看到一个类有 `virtual` 函数，析构就要 `virtual`——把这条当反射写。
:::

::: callout 3. 子类函数没加 override
```cpp
class Dog : public Animal {
public:
    void Speak() const {  // 注意大小写！
    }
};
```
没加 `override`，编译器把它当"新函数"处理，多态根本没生效。**所有覆盖父类的函数都加 `override`**——拼错时立刻编译报错。
:::

::: callout 4. 在构造/析构里调虚函数
```cpp
class Base {
public:
    Base() { foo(); }            // 这里调的是 Base::foo，不是子类的
    virtual void foo() { ... }
};
```
构造和析构期间，对象还不是"完整的子类"，虚函数会调到当前层的版本而不是最终子类的版本。**别在构造析构里调虚函数**。
:::

::: callout 5. 把所有数据都设成 public
完全没封装的类等于 `struct`，业务规则保不住。**默认 `private`，需要外界用的接口才放 `public`**。
:::
