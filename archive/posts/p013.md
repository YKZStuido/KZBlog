## 三个容器一句话总结

写代码经常遇到"我有一堆东西，想把它们存起来再处理"。C++ 给了三个最常用的容器：

- **`std::vector`**：会自己变长的数组。**最常用**，能随机访问、能末尾追加。
- **`std::stack`**：栈，**后进先出**。最近放进去的最先拿出来。
- **`std::queue`**：队列，**先进先出**。先放进去的先拿出来。

形象点：

- **vector** 像一排能往后加格子的储物柜。
- **stack** 像叠起来的盘子——最上面那个先拿走。
- **queue** 像排队买饭——先来的先打饭。

下面一个一个讲。

## std::vector：万金油容器

要 `#include <vector>`。

### 创建和初始化

```cpp
#include <vector>

std::vector<int> a;                     // 空 vector
std::vector<int> b(5);                  // 5 个元素，都是 0
std::vector<int> c(5, 99);              // 5 个 99
std::vector<int> d{1, 2, 3, 4, 5};      // 列表初始化
std::vector<int> e = {10, 20, 30};      // 等价写法
```

`std::vector<int>` 表示"装 int 的 vector"。**尖括号里写元素类型**：

```cpp
std::vector<double> nums;
std::vector<std::string> names;
std::vector<Student> students;          // 装结构体也行
std::vector<std::vector<int>> grid;     // 二维：vector 里装 vector
```

### 添加元素

```cpp
std::vector<int> v;
v.push_back(10);        // 末尾追加 10
v.push_back(20);        // 末尾追加 20
v.push_back(30);        // 末尾追加 30
// v 现在是 {10, 20, 30}
```

`push_back` 是最常用的添加方式——**永远在末尾**。

要在中间插入用 `insert`：

```cpp
std::vector<int> v{1, 2, 4, 5};
v.insert(v.begin() + 2, 3);     // 在下标 2 的位置插入 3
// v 现在是 {1, 2, 3, 4, 5}
```

> 在中间插入比较慢——后面所有元素都要往后挪一格。**vector 的强项是末尾追加，不是中间插**。

### 访问元素

```cpp
std::vector<int> v{10, 20, 30, 40, 50};

std::cout << v[0] << "\n";       // 10  下标从 0 开始
std::cout << v[2] << "\n";       // 30
std::cout << v.front() << "\n";  // 10  第一个
std::cout << v.back() << "\n";   // 50  最后一个
std::cout << v.size() << "\n";   // 5   元素个数

// v[100];                       // 越界！未定义行为，可能崩可能读垃圾
v.at(100);                       // 越界会抛异常，至少崩得明确
```

### 删除元素

```cpp
std::vector<int> v{10, 20, 30, 40, 50};

v.pop_back();                    // 删末尾，v 现在是 {10, 20, 30, 40}
v.erase(v.begin());              // 删第一个，v 现在是 {20, 30, 40}
v.erase(v.begin() + 1);          // 删下标 1 的，v 现在是 {20, 40}
v.clear();                       // 全部清空
```

### 遍历的三种写法

```cpp
std::vector<int> v{1, 2, 3, 4, 5};

// 1. 范围 for（最现代、最推荐）
for (int x : v) {
    std::cout << x << " ";
}

// 2. 下标循环（要修改元素或要用下标时）
for (size_t i = 0; i < v.size(); ++i) {
    v[i] *= 2;
}

// 3. 迭代器（写底层的算法时才用，平时少用）
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it << " ";
}
```

::: callout 想边遍历边修改？加个 `&`
```cpp
for (int& x : v) {           // 注意 &
    x *= 2;                  // 真的改 v 里的元素
}
```
不加 `&`，`x` 是元素的拷贝，改了不影响 v。**这是新手最常踩的坑之一**。
:::

### 二维 vector：表格 / 棋盘

```cpp
// 一个 3 行 4 列、全是 0 的二维数组
std::vector<std::vector<int>> grid(3, std::vector<int>(4, 0));

grid[0][0] = 1;
grid[1][2] = 5;

for (const auto& row : grid) {
    for (int x : row) {
        std::cout << x << " ";
    }
    std::cout << "\n";
}
```

写迷宫、棋盘、任何"行 × 列"的数据都用这个。

### 一个完整例子：班级分数

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> scores;

    // 输入
    int n;
    std::cout << "几个人？";
    std::cin >> n;
    for (int i = 0; i < n; ++i) {
        int s;
        std::cin >> s;
        scores.push_back(s);
    }

    // 排序（从高到低）
    std::sort(scores.begin(), scores.end(), std::greater<int>());

    // 求和、平均、最高、最低
    int sum = 0;
    for (int s : scores) sum += s;

    std::cout << "排序后：";
    for (int s : scores) std::cout << s << " ";
    std::cout << "\n";
    std::cout << "平均：" << double(sum) / n << "\n";
    std::cout << "最高：" << scores.front() << "\n";
    std::cout << "最低：" << scores.back() << "\n";
}
```

50% 的入门代码就长这样——**vector + 循环 + 算法**。

## std::stack：后进先出

要 `#include <stack>`。

### 形象理解

想象洗碗时叠起来的盘子——你只能从**最上面**放、从**最上面**拿。最后放上去的盘子最先被拿走。

```
拿 →  [盘 D]   ← 最后放的，最先拿
      [盘 C]
      [盘 B]
放 →  [盘 A]   ← 最早放的
```

这就是 **LIFO**（Last In, First Out）。

### 基本操作

栈只有四个核心操作：

```cpp
#include <stack>

std::stack<int> st;

st.push(1);              // 压栈：放最上面
st.push(2);
st.push(3);

std::cout << st.top();   // 看最上面，3
st.pop();                // 弹栈：拿走最上面
std::cout << st.top();   // 现在最上面是 2
std::cout << st.size();  // 2
std::cout << st.empty(); // false（0）
```

::: callout `pop()` 不返回值
和很多教材教的不一样——`std::stack::pop()` **只删，不返回**。要拿到那个值得先 `top()`：
```cpp
int x = st.top();
st.pop();
```
为什么这么设计？历史包袱 + 异常安全考虑。**记住这个细节**就行。
:::

### 经典应用：括号匹配

判断一段表达式里 `( ) [ ] { }` 配不配对——**栈的标准用法**：

```cpp
#include <iostream>
#include <stack>
#include <string>

bool is_balanced(const std::string& s) {
    std::stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            if (st.empty()) return false;
            char top = st.top();
            st.pop();
            if (c == ')' && top != '(') return false;
            if (c == ']' && top != '[') return false;
            if (c == '}' && top != '{') return false;
        }
    }
    return st.empty();
}

int main() {
    std::cout << is_balanced("([{}])") << "\n";    // 1（true）
    std::cout << is_balanced("([)]")   << "\n";    // 0（false）
    std::cout << is_balanced("(((")    << "\n";    // 0
}
```

**思路**：左括号入栈，右括号来了就检查栈顶——配对就弹出，不配对就 false。最后栈得是空的。

栈最适合处理这种**"嵌套结构"**——计算器表达式、HTML 标签、目录层级、函数调用栈本身……都是这个模式。

### 另一个例子：反转字符串

```cpp
#include <iostream>
#include <stack>
#include <string>

int main() {
    std::string s = "hello";
    std::stack<char> st;
    for (char c : s) st.push(c);

    std::string reversed;
    while (!st.empty()) {
        reversed += st.top();
        st.pop();
    }
    std::cout << reversed << "\n";    // olleh
}
```

把字符全压进栈，再一个个弹出来——天然就反过来了。

## std::queue：先进先出

要 `#include <queue>`。

### 形象理解

排队买饭——**先来的先打饭，后来的等后面**。

```
出 ←  [小明][小红][小李][小张]  ← 入
       (front)            (back)
```

这就是 **FIFO**（First In, First Out）。

### 基本操作

```cpp
#include <queue>

std::queue<int> q;

q.push(1);                // 入队：从队尾加入
q.push(2);
q.push(3);

std::cout << q.front();   // 看队首，1（最早进的）
std::cout << q.back();    // 看队尾，3（最晚进的）
q.pop();                  // 出队：拿走队首
std::cout << q.front();   // 2

std::cout << q.size();    // 2
std::cout << q.empty();   // false
```

和栈类似，`pop()` 也**不返回值**——要先 `front()`。

### 经典应用：任务调度

模拟"打印机排队"——先送来的任务先打：

```cpp
#include <iostream>
#include <queue>
#include <string>

int main() {
    std::queue<std::string> tasks;

    tasks.push("打印作业.docx");
    tasks.push("打印照片.jpg");
    tasks.push("打印简历.pdf");

    while (!tasks.empty()) {
        std::cout << "正在打印：" << tasks.front() << "\n";
        tasks.pop();
    }
    // 输出顺序就是入队顺序
}
```

凡是"按到达顺序处理"的场景——网络消息、用户请求、广播通知——都是队列的舞台。

### 另一个例子：统计数字流的"近 K 个平均"

```cpp
#include <iostream>
#include <queue>

int main() {
    std::queue<int> window;
    int K = 3;
    int sum = 0;

    int nums[] = {10, 20, 30, 40, 50, 60};
    for (int x : nums) {
        window.push(x);
        sum += x;
        if (window.size() > K) {
            sum -= window.front();
            window.pop();
        }
        if (window.size() == K) {
            std::cout << "近 3 个平均：" << double(sum) / K << "\n";
        }
    }
}
```

队列正好做"滑动窗口"——超过 K 个就把最早进来的踢掉。

## 三者对比

**`std::vector`：**
- 加在哪：末尾 O(1)、中间 O(n)。
- 删哪个：末尾 / 中间任意。
- 能随机访问吗：能，`v[i]` 直接拿。
- 何时用：**默认选择**，不知道用哪个就用它。

**`std::stack`：**
- 加在哪：只能加在顶部。
- 删哪个：只能删顶部。
- 能随机访问吗：不能，只能看 `top()`。
- 何时用：嵌套结构、回溯、"最近的优先"。

**`std::queue`：**
- 加在哪：只能加在队尾。
- 删哪个：只能删队首。
- 能随机访问吗：不能，只能看 `front()` 或 `back()`。
- 何时用：按到达顺序处理、一层一层扩展（下一篇广搜就用它）。

**一句话决策**：

- 不知道用哪个 → **vector**。
- 处理"嵌套 / 反转 / 撤销 / 最近的优先" → **stack**。
- 处理"按到达顺序 / 一层层扩展" → **queue**。

::: callout stack 和 queue 其实是 vector / deque 的封装
`std::stack` / `std::queue` 是**容器适配器**——内部默认用 `std::deque` 实现，给你提供一组限制了的接口（只能操作一端）。

为什么不直接用 vector？**用 stack / queue 是在给读者一个语义信号**——"我在这里只会做后进先出 / 先进先出的操作，请按这个理解"。代码意图比省那一行 include 重要。
:::

## 综合例子：用栈实现简单计算器

支持加减乘除、有优先级的小型计算器（中缀表达式求值）——只用 vector / stack 就能写出来：

```cpp
#include <iostream>
#include <stack>
#include <string>
#include <cctype>

int precedence(char op) {
    if (op == '+' || op == '-') return 1;
    if (op == '*' || op == '/') return 2;
    return 0;
}

int apply(int a, int b, char op) {
    switch (op) {
        case '+': return a + b;
        case '-': return a - b;
        case '*': return a * b;
        case '/': return a / b;
    }
    return 0;
}

int evaluate(const std::string& expr) {
    std::stack<int> nums;       // 数字栈
    std::stack<char> ops;       // 运算符栈

    for (size_t i = 0; i < expr.size(); ++i) {
        char c = expr[i];
        if (std::isspace(c)) continue;

        if (std::isdigit(c)) {
            int n = 0;
            while (i < expr.size() && std::isdigit(expr[i])) {
                n = n * 10 + (expr[i] - '0');
                ++i;
            }
            --i;
            nums.push(n);
        } else if (c == '+' || c == '-' || c == '*' || c == '/') {
            // 把前面优先级 >= 当前的先算掉
            while (!ops.empty() && precedence(ops.top()) >= precedence(c)) {
                int b = nums.top(); nums.pop();
                int a = nums.top(); nums.pop();
                char op = ops.top(); ops.pop();
                nums.push(apply(a, b, op));
            }
            ops.push(c);
        }
    }
    // 剩下的全算完
    while (!ops.empty()) {
        int b = nums.top(); nums.pop();
        int a = nums.top(); nums.pop();
        char op = ops.top(); ops.pop();
        nums.push(apply(a, b, op));
    }
    return nums.top();
}

int main() {
    std::cout << evaluate("3 + 5 * 2")       << "\n";   // 13
    std::cout << evaluate("10 - 4 * 2 + 1")  << "\n";   // 3
    std::cout << evaluate("100 / 4 / 5")     << "\n";   // 5
}
```

这段代码里：

- **数字栈**装待计算的操作数。
- **运算符栈**装等待执行的运算符。
- 遇到优先级更高或相同的运算符就先算前面的——经典栈应用。

**别一行行死记**——这是个值得看几遍、亲手敲一遍、再自己改改（加上括号支持？支持负数？）的练习题。

## 常见坑

::: callout 1. 遍历 vector 时用 push_back
```cpp
for (int x : v) {
    if (x > 0) v.push_back(x);    // 危险！
}
```
`push_back` 可能让 vector 重新分配内存，**正在遍历的迭代器全部失效**——结果可能崩溃可能死循环。**遍历时不要修改容器结构**，要改就先记下来再统一加。
:::

::: callout 2. 空容器调 top() / front() / pop()
```cpp
std::stack<int> st;
st.top();      // UB（未定义行为），可能崩可能拿到垃圾
```
**先判断 `!st.empty()`** 再 top/pop。
:::

::: callout 3. `pop()` 不返回值
```cpp
int x = st.pop();    // 编译错：pop() 返回 void
```
正确写法：
```cpp
int x = st.top();
st.pop();
```
:::

::: callout 4. 范围 for 没加 `&` 改不了原值
```cpp
std::vector<int> v{1, 2, 3};
for (int x : v) x *= 2;          // v 还是 {1,2,3}！

for (int& x : v) x *= 2;         // v 变成 {2,4,6}
```
要改原值就**加 `&`**。只读建议 `const auto&`（避免拷贝）。
:::

::: callout 5. vector 越界访问
```cpp
std::vector<int> v(5);
v[10] = 99;            // UB，能编译能跑，但会写到不该写的地方
```
`v[i]` **不做边界检查**。担心越界用 `v.at(i)`——越界会抛 `std::out_of_range`。
:::
