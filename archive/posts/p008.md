## STL 是什么

**STL（Standard Template Library）** 是 C++ 标准库的一部分，由三大件组成：

- **容器（Container）**：装数据的——`vector`、`map`、`set` 等。
- **迭代器（Iterator）**：遍历容器的统一接口，可以理解为"能 ++ 的指针"。
- **算法（Algorithm）**：作用在迭代器区间上的通用算法——排序、查找、累加……写在 `<algorithm>` 头里。

三者通过迭代器解耦——同一个 `std::sort` 既能排 `vector` 也能排数组，因为它只看迭代器，不关心容器实现。

::: callout 为什么叫"模板"库
`std::vector<int>`、`std::vector<std::string>`、`std::vector<MyClass>` 是同一份代码，靠**模板**对不同类型重新生成。模板的细节这个合集后面会单独讲，**这篇只用，不深究**。
:::

## 序列容器

序列容器按"插入顺序"存元素，元素之间有先后。

### `std::vector`：动态数组（首选）

之前几篇已经反复出场了。再列一遍最常用的接口：

```cpp
#include <vector>

std::vector<int> v;            // 空
std::vector<int> v2(5, 0);     // 5 个 0
std::vector<int> v3{1, 2, 3};  // 列表初始化

v.push_back(10);               // 末尾追加
v.pop_back();                  // 末尾删除

v[0];                          // 下标访问，不检查越界
v.at(0);                       // 带边界检查，越界抛异常
v.front();                     // 第一个
v.back();                      // 最后一个

v.size();                      // 元素个数
v.empty();                     // 是否为空
v.clear();                     // 清空

v.insert(v.begin() + 2, 99);   // 在第 3 位插入 99
v.erase(v.begin() + 2);        // 删第 3 位
```

::: callout vector 的扩容代价
`push_back` 触发扩容时会**整体搬到新内存**——已经满了就重新申请一块更大的，复制过去。摊销下来仍然是 O(1)，但如果你**预先知道大概多少元素**，先 `v.reserve(n)` 一下能避免多次重新分配，性能差距挺大。
:::

### `std::array`：定长数组

C 风格数组的现代替代，长度在**编译期**确定：

```cpp
#include <array>

std::array<int, 5> a{1, 2, 3, 4, 5};
a[0];                  // 1
a.size();              // 5（永远不变）
```

比 `int arr[5]` 强在：能传给函数不衰变成指针、有 `size()`、能当 STL 容器用。**长度固定且已知**就用 `std::array`，需要动态长度才用 `std::vector`。

### `std::deque`：双端队列

两头都能 O(1) 插删。用得没 `vector` 多，但需要 `push_front` 时它就是答案：

```cpp
#include <deque>

std::deque<int> dq;
dq.push_back(1);
dq.push_front(0);      // vector 没这个
```

### `std::list`：双向链表

每个节点单独分配，能在中间任意位置 O(1) 插删。**但缓存不友好、没有随机访问**——实际工程里用得很少。**入门阶段了解即可**，需要"中间频繁插删"的场景考虑 `std::list`，其余一律 `std::vector`。

## 关联容器

按 key 查找，不按插入顺序。

### `std::map`：有序键值对

红黑树实现，**按 key 自动排序**，查找/插入/删除都是 O(log n)：

```cpp
#include <map>

std::map<std::string, int> ages;
ages["亚柯洲"] = 13;
ages["旺财"] = 5;
ages["咪咪"] = 3;

std::cout << ages["亚柯洲"] << "\n";   // 13

if (ages.count("路人")) {
    // 不会进来
}

// 遍历——按 key 字典序
for (const auto& [name, age] : ages) {
    std::cout << name << " → " << age << "\n";
}
```

`for (const auto& [name, age] : ages)` 是 C++17 的**结构化绑定**——把 pair 拆成两个变量，超好用。

::: callout `map[key]` 会"创建"
`ages["不存在的人"]` 这一下，**会在 map 里插入一个值为默认值（int 是 0）的项**——可能不是你想要的。

只想"查"不想"插"，用 `find`：
```cpp
auto it = ages.find("路人");
if (it != ages.end()) {
    std::cout << it->second << "\n";
}
```
或者用 `count`（返回 0 / 1）。
:::

### `std::set`：有序集合

存"唯一的、自动排序的"元素：

```cpp
#include <set>

std::set<int> s;
s.insert(3);
s.insert(1);
s.insert(4);
s.insert(1);      // 重复，被忽略

for (int x : s) std::cout << x << " ";   // 1 3 4

if (s.count(3)) {
    std::cout << "3 在里面\n";
}
```

### `std::unordered_map` / `std::unordered_set`：哈希版

哈希表实现，**不排序**，但平均 O(1) 查找——不需要"按顺序遍历"时优先用这个：

```cpp
#include <unordered_map>

std::unordered_map<std::string, int> counter;
counter["a"]++;
counter["a"]++;
counter["b"]++;
// counter["a"] == 2, counter["b"] == 1
```

::: callout 选 map 还是 unordered_map
- **要按 key 顺序遍历** / 需要范围查询（"找 ≥ k 的所有元素"）→ `std::map`。
- **只要查得快、不在乎顺序** → `std::unordered_map`，**通常更快**。

入门阶段大部分计数、查表场景用 `unordered_map` 就行。
:::

## 容器适配器

基于已有容器封装出特定接口。

### `std::stack`：栈，后进先出

```cpp
#include <stack>

std::stack<int> st;
st.push(1);
st.push(2);
st.push(3);

std::cout << st.top() << "\n";    // 3
st.pop();
std::cout << st.top() << "\n";    // 2
```

注意 `pop()` 不返回值——要先 `top()` 再 `pop()`。

### `std::queue`：队列，先进先出

```cpp
#include <queue>

std::queue<int> q;
q.push(1);
q.push(2);
q.push(3);

std::cout << q.front() << "\n";   // 1
q.pop();
std::cout << q.front() << "\n";   // 2
```

### `std::priority_queue`：优先队列（堆）

每次 `top()` 拿到的是**当前最大**的元素：

```cpp
#include <queue>

std::priority_queue<int> pq;
pq.push(3);
pq.push(1);
pq.push(4);
pq.push(1);
pq.push(5);

while (!pq.empty()) {
    std::cout << pq.top() << " ";
    pq.pop();
}
// 5 4 3 1 1
```

要小顶堆（每次拿最小）：

```cpp
std::priority_queue<int, std::vector<int>, std::greater<int>> pq;
```

模板参数有点丑，但用一次就熟了。

## 迭代器：容器的"指针"

每个 STL 容器都有 `begin()` 和 `end()`，返回**迭代器**——指向第一个元素和"末尾后一位"。

```cpp
std::vector<int> v{10, 20, 30};
auto it = v.begin();             // 指向 10
std::cout << *it << "\n";        // 解引用，10

++it;                            // 移动到 20
std::cout << *it << "\n";        // 20

it = v.end();                    // "末尾后一位"，不能解引用
```

::: callout `end()` 不指向最后一个元素
`end()` 是**最后一个元素的下一位**——一个用来表示"到此为止"的哨兵。所以遍历的标准写法是：

```cpp
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it << " ";
}
```

注意是 `!=` 不是 `<`——并不是所有迭代器都支持比大小。
:::

迭代器是 STL 的"通用语言"——所有算法都接收迭代器，所以同一个算法能用在任何容器上。

## `<algorithm>`：现成轮子

`<algorithm>` 头里有 100+ 个函数。下面是**入门阶段最值得记住的几个**。

### 排序

```cpp
#include <algorithm>
#include <vector>

std::vector<int> v{3, 1, 4, 1, 5, 9, 2, 6};
std::sort(v.begin(), v.end());
// 1 1 2 3 4 5 6 9

std::sort(v.begin(), v.end(), std::greater<int>());
// 9 6 5 4 3 2 1 1

std::sort(v.begin(), v.end(), [](int a, int b) {
    return std::abs(a - 5) < std::abs(b - 5);  // 离 5 越近排越前
});
```

`std::sort` 是 introsort，平均 O(n log n)，没特殊需求就用它。

### 查找

```cpp
auto it = std::find(v.begin(), v.end(), 4);
if (it != v.end()) {
    std::cout << "找到了，下标 " << it - v.begin() << "\n";
}
```

容器有序时用**二分查找**：

```cpp
std::sort(v.begin(), v.end());
bool exists = std::binary_search(v.begin(), v.end(), 4);
auto lb = std::lower_bound(v.begin(), v.end(), 4);  // 第一个 >= 4 的位置
auto ub = std::upper_bound(v.begin(), v.end(), 4);  // 第一个 > 4 的位置
```

### 求和、最值、计数

```cpp
#include <numeric>

int sum = std::accumulate(v.begin(), v.end(), 0);
int prod = std::accumulate(v.begin(), v.end(), 1, std::multiplies<int>());

auto mn = std::min_element(v.begin(), v.end());   // 迭代器
auto mx = std::max_element(v.begin(), v.end());
std::cout << *mn << " " << *mx << "\n";

int n4 = std::count(v.begin(), v.end(), 4);
int even = std::count_if(v.begin(), v.end(), [](int x){ return x % 2 == 0; });
```

### 遍历与变换

```cpp
std::for_each(v.begin(), v.end(), [](int x){
    std::cout << x << " ";
});

// 把每个元素变成它的平方
std::transform(v.begin(), v.end(), v.begin(), [](int x){
    return x * x;
});
```

### 删除：erase-remove 惯用法

`std::remove` 不真删——它只是把"该删的"挪到末尾，返回新的逻辑结尾。**真的删要再调用 erase**：

```cpp
std::vector<int> v{1, 2, 3, 2, 4, 2, 5};

v.erase(std::remove(v.begin(), v.end(), 2), v.end());
// 1 3 4 5
```

C++20 以后可以直接 `std::erase(v, 2)`，简洁很多。但 `erase-remove` 这个惯用法**你一定会在老代码里反复看到**。

### 反转、去重、生成

```cpp
std::reverse(v.begin(), v.end());

// 去重前必须先排序
std::sort(v.begin(), v.end());
v.erase(std::unique(v.begin(), v.end()), v.end());

std::vector<int> nums(10);
std::iota(nums.begin(), nums.end(), 1);   // 1, 2, 3, ..., 10
```

## `std::string`：本质是个容器

`std::string` 其实就是 `std::vector<char>` 的特化变体——一切容器接口它都有：

```cpp
std::string s = "hello world";
s.size();                           // 11
s[0];                               // 'h'
s.substr(6, 5);                     // "world"
s.find("world");                    // 6（找不到返回 std::string::npos）

// 查替换
size_t pos = s.find("world");
if (pos != std::string::npos) {
    s.replace(pos, 5, "C++");
}
// s == "hello C++"

// 拼接
std::string a = "abc";
std::string b = "def";
std::string c = a + b;              // "abcdef"
a += b;                             // a 现在是 "abcdef"
```

## 容器到底用哪个？速查

入门阶段记住这个就够：

- 一组同类型的东西、需要快速末尾追加 → **`std::vector`**
- 长度固定且编译期已知 → **`std::array`**
- 键值对，要查得快、不在乎顺序 → **`std::unordered_map`**
- 键值对，要按 key 顺序遍历 → **`std::map`**
- 集合（去重 + 快速判存在）→ **`std::unordered_set`**
- 后进先出 / 先进先出 → **`std::stack` / `std::queue`**
- 反复取最大值 → **`std::priority_queue`**

> **80/20 原则**：80% 的入门代码 `std::vector` + `std::unordered_map` + `std::string` 就能搞定。

## 一个综合例子：词频统计

输入一段文本，统计每个单词出现的次数，按出现次数从多到少打印前 10：

```cpp
#include <iostream>
#include <vector>
#include <unordered_map>
#include <string>
#include <algorithm>
#include <sstream>

int main() {
    std::string text =
        "to be or not to be that is the question "
        "whether tis nobler in the mind to suffer "
        "the slings and arrows of outrageous fortune "
        "or to take arms against a sea of troubles";

    // 1. 切词 + 计数
    std::unordered_map<std::string, int> freq;
    std::istringstream iss(text);
    std::string word;
    while (iss >> word) {
        freq[word]++;
    }

    // 2. 倒进 vector 方便排序
    std::vector<std::pair<std::string, int>> items(freq.begin(), freq.end());

    // 3. 按出现次数从多到少排
    std::sort(items.begin(), items.end(), [](const auto& a, const auto& b) {
        return a.second > b.second;
    });

    // 4. 打印前 10
    int top = std::min<int>(10, items.size());
    for (int i = 0; i < top; ++i) {
        std::cout << items[i].first << " " << items[i].second << "\n";
    }
    return 0;
}
```

短短二十几行，覆盖了 `unordered_map`、`vector`、`pair`、`sort` + lambda、`istringstream`、范围 `auto` ——**这就是写 STL 该有的密度**。换 Python 写也就这么多行。

## 常见坑

::: callout 1. 迭代器失效
```cpp
std::vector<int> v{1, 2, 3, 4};
for (auto it = v.begin(); it != v.end(); ++it) {
    if (*it == 2) v.push_back(99);   // ← 危险
}
```
`push_back` 可能触发扩容，把所有迭代器作废，循环就乱套了。**遍历容器时不要修改容器结构**——要改就先记下来再统一处理。
:::

::: callout 2. `map[key]` 意外插入
```cpp
if (m["不存在的 key"] == 0) { ... }   // 已经插入了 key！
```
查存在用 `m.count(k)` 或 `m.find(k)`，别用 `[]`。
:::

::: callout 3. `vector::erase` 后接着用旧迭代器
```cpp
auto it = v.begin();
v.erase(it);
*it;     // 旧 it 已失效，UB
```
`erase` 返回**指向被删元素之后**的新迭代器，要用就用它返回的：
```cpp
it = v.erase(it);
```
:::

::: callout 4. `std::sort` 的比较函数返回 `<=`
比较函数必须是**严格弱序**——`a < b` 返回 true 时，`b < a` 必须返回 false。写成 `<=` 在 `a == b` 时两边都返回 true，违反规则，触发未定义行为，运行结果千奇百怪。**用 `<` 或 `>`**。
:::

::: callout 5. `unordered_map` 没法用自定义类型当 key
```cpp
std::unordered_map<MyClass, int> m;   // 编译错
```
默认没有 hash 函数。要么提供一个 `std::hash<MyClass>` 的特化，要么改用 `std::map`（只需要 `<` 运算符）。
:::
